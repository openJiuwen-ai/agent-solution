# 开发快速入门

本文档将指导你理解 EDPAgent 的配置体系，并通过理财场景示例快速上手 Governance 配置和 Skill 开发。

## 前置条件

| 组件 | 要求 |
|------|------|
| JDK | 17+ |
| Maven | 3.8+ |
| Python | 3.10+ |
| Redis | 5.0+（本地或Docker） |
| 大模型 API Key | DeepSeek 或其他 OpenAI 兼容模型 |

## 核心理念

EDPAgent 的设计哲学是**配置驱动**：

> 角色定义、行为边界、工具约束、业务话术、技能路由——全部通过 YAML 配置声明，无需修改 Java 代码。

配置体系分为三层：

1. **planrule.yaml** - 定义 Agent 是谁、负责什么、不能做什么、遵循什么协议
2. **actrule.yaml** - 定义执行约束：最大步数、允许的工具、工具调用限制、Skill加载模式
3. **scriptconfig.yaml** - 定义话术模板：通用状态话术、思维链话术、ask_user模板

每个业务场景可以在自己的目录下放置同名 YAML 文件，进行**增量覆盖**。

## 步骤一：准备本地开发环境

### 1. 启动 Redis

```bash
docker run -d --name edp-redis -p 6379:6379 redis:7-alpine
```

### 2. 配置环境变量

在代码仓根目录创建 `.env` 文件或直接设置环境变量：

```bash
# 必填：大模型 API Key
export EDP_AGENT_MODEL_API_KEY=your-deepseek-api-key

# Redis 配置（本地默认即可）
export EDPA_REDIS_HOST=localhost
export EDPA_REDIS_PORT=6379

# 使用理财演示场景
export EDP_AGENT_SCENARIO_HOME=../scenarios/wealth-demo
```

### 3. 启动服务

```bash
cd engine
./mvnw spring-boot:run
```

或者在 IDE 中直接运行 `EdpaApplication.java` 主类。

### 配置覆盖策略

| 覆盖方式 | 适用字段 | 说明 |
|----------|----------|------|
| 替代式覆盖 | `scope`、`supplementary_prompt` | 场景配置完全替代框架默认值 |
| 继承式覆盖 | `role`、`max_subtasks`、`max_steps` | 场景有值则覆盖，无值则继承框架 |
| 叠加合并 | `allowed_tools` | 场景工具列表追加到框架列表 |

### 理财场景 planrule 示例解读

```yaml
planrule:
  scenario_name: "理财购买"
  scenario_description: "理财产品推荐、筛选、购买全流程"

  # 替代式覆盖：业务范围
  scope:
    allowed: "理财产品推荐、筛选、购买、银行账户余额查询、银行账户间转账..."
    denied: "基金相关业务、股票相关业务、保险相关业务..."

  # 追加式：场景特有规则
  supplementary_prompt:
    additional_prompt: |
      ## 理财场景强制规则
      - 业务范围内请求必须调用 todo_create，禁止直接文本回答
      - 每次只执行一个 tool_call，禁止并发
      - 优先使用理财场景预定义 catalog_id

  # Skill 路由
  skill_routing:
    - trigger: "用户首次请求推荐理财产品"
      skill: "product_recommend_skill"
      priority: 1
```

## 步骤三：理解 Skill 结构

以理财场景的 [product_recommend_skill](file:///D:/AgentTool/EDPAgent_java/branch_edpa_java/agent-store/edp-agent-java/scenarios/wealth-demo/skills/product_recommend_skill/) 为例：

```
product_recommend_skill/
├── SKILL.yaml          # Skill 元数据（必填）
├── SKILL.md            # Skill 使用说明（给 Agent 参考）
└── scripts/
    └── run_product_recommend_skill.py  # Python 执行脚本
```

### SKILL.yaml 核心字段

```yaml
name: product_recommend_skill
description: "根据用户理财意图进入理财推荐流程并展示推荐产品列表。触发词：推荐理财产品、帮我看看理财、有什么理财可以买。不要用于：产品选择确认、资金筹划、账户查询。"
scripts:
  product_recommend_success: "我找到以上您可能感兴趣的产品，可以告诉我购买哪支产品及购买金额。"
  product_recommend_empty: '我理解你对稳健收益的追求，但"稳赚不赔"的产品在金融领域并不存在。'
```

> 💡 **关键**：`description` 字段直接作为 System Prompt 发给 LLM，决定了 Skill 何时被触发。写好 description 是 Skill 开发的核心。

## 步骤四：调用测试服务

服务启动后（默认端口 8190），发送一个理财推荐请求：

```bash
curl -N -X POST http://localhost:8190/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "test-wealth-001",
    "method": "sendStreamingMessage",
    "params": {
      "message": {
        "messageId": "msg-001",
        "role": "user",
        "parts": [{"kind": "text", "text": "帮我推荐一些理财产品"}]
      }
    }
  }'
```

观察 SSE 事件流，你将看到：
1. 规划阶段：Agent 创建 Todo 任务列表
2. 执行阶段：调用 Skill/Versatile 获取推荐结果
3. 总结阶段：输出最终回答
