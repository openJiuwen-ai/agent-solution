1. 概述

jiuwenbox 是运行在 Linux 上的沙箱管理服务。 对外暴露 FastAPI HTTP 管理面（默认 :8321）；每个沙箱实例通过 bubblewrap（bwrap） 创建独立命名空间与挂载视图，并在沙箱内启动 长期驻留的 sandbox-daemon。

SandboxManager、ProcessRuntime 是 box-server 进程内的对象，不是独立 OS 进程。 隔离底座 bwrap（bubblewrap） 是 Flatpak 维护的轻量沙箱启动器：用 namespace + 挂载 + 降权拼出子环境； jiuwenbox 将 YAML 策略转为 bwrap 参数（supervisor/bwrap.py）。

关键设计：bwrap 的昂贵初始化每个沙箱只做一次，后续 exec、文件读写走 daemon 的 Unix Domain Socket IPC，子进程继承同一套 Landlock / seccomp / cgroup 约束，避免「每条命令起一个全新沙箱」。

宿主机侧（可信）

jiuwenbox-server、SandboxManager、ProcessRuntime、控制 socket listener、cgroup/netns 管理。

沙箱侧（不可信载荷）

bwrap 子树内的 landlock-launcher → sandbox-daemon（PID 1）→ 用户 fork+exec 命令。

宿主机组件 沙箱内组件 IPC / 控制通道

3. 进程间关系详解

3.1 宿主机全局进程树

jiuwenbox-server (box-server, 主进程)
├── [线程] FastAPI 事件循环 + HTTP
├── [线程池] 阻塞 IPC（exec / read_file 等）
├── [可选] zombie reaper（SIGCHLD 或定时 poll）
│
├── 沙箱 A
│   ├── bwrap monitor          ← Popen 直接子进程
│   ├── [短暂] userns helper   ← bwrap 内部，也是 box-server 子进程
│   └── [isolated] ip netns exec jbx-A → bwrap
│
├── 沙箱 B …
│
└── exec_background 任务
    └── 每次任务再起一个 bwrap   ← box-server 直接子进程

box-server 启动时设 PR_SET_CHILD_SUBREAPER，回收 bwrap monitor、userns helper 等僵尸进程。

3.2 父子与 IPC 关系表

关系类型说明
box-server → bwrap monitor父子（Popen）pass_fds 传入 control listener、seccomp memfd；preexec 加入 cgroup
launcher → daemon同进程（PID 1）Landlock 后 compile/exec，无二次 execve 读盘
daemon → 用户命令父子（fork+exec）start_new_session=True，防 kill(0) 误杀 daemon
box-server ↔ daemonUDS 客户端/服务端非父子；socket 在宿主机，用户代码不可见
Adapter/EDP ↔ jiuwenboxHTTP 客户端独立容器进程，无直接父子

3.3 两类执行路径

路径 A — 同步 exec / 文件 API（热路径）

HTTP → box-server → 线程池
  → connect(control.sock)
  → sandbox-daemon → fork+exec

不起新 bwrap。upload/download/files 在 daemon 进程内完成。

路径 B — exec_background

box-server → Popen(新 bwrap + 命令)

每任务独立 bwrap 树，与 daemon 并行，是 box-server 直接子进程。

4. 沙箱生命周期

provisioning → ready ⇄ stopped → error / deleting
阶段含义可执行操作
provisioning正在建 netns、cgroup、bind、起 bwrap等待就绪
readydaemon 已 accept 控制 fdexec · upload · download · files · search
stopped已 stop，资源可能仍保留start / restart
error创建或运行失败delete / restart
deleting销毁中—
沙箱状态主要在服务进程内存中维护；jiuwenbox-server 重启后不会从磁盘「复活」旧沙箱实例。 EDPAgent 重启若创建新 sandbox，Adapter 需重新解析 sandbox_id。

5.3 并发与恢复

  • FastAPI 路由为 async，daemon IPC 在线程池执行。
  • exec 有并发信号量限流；文件 IPC 通常不限流。
  • IPC 超时或连接失败时，SandboxManager 可自动 restart_sandbox 并重试一次。

7. 环境隔离（六层防护）

隔离由 SecurityPolicy（YAML）驱动，在 supervisor/bwrap.py 转为 bwrap 参数，并与 Landlock / seccomp / cgroup 叠加。

Layer 6
网络隔离（isolated）
独立 netns + veth；netns 内 iptables 做 egress/ingress 域名/IP/端口白名单（默认 deny）。宿主机 iptables 阻止沙箱访问 jiuwenbox 管理端口。
Layer 5
Cgroup 资源限额
宿主机写 /sys/fs/cgroup/jiuwenbox/{id}/：memory_max、cpu_max、pids_max。子进程经 preexec 加入 cgroup，沙箱内无法自行放宽。
Layer 4
Seccomp BPF
按架构生成 BPF，经 bwrap --seccomp 加载。拦截危险 syscall；额外阻止对 PID 1/2、进程组、broadcast 的 kill，防止沙箱代码杀掉 daemon。
Layer 3
Landlock 路径白名单
launcher 在跑用户代码前 prctl(NO_NEW_PRIVS) + landlock_restrict_self。白名单来自 read_only / read_write / bind_mounts 等。/jiuwenbox 为保留路径，用户策略无法加回。
Layer 2
Bubblewrap 挂载视图
--ro-bind / --bind / --tmpfs 构造沙箱可见 FS；--proc / --dev；默认 UID/GID = nobody(65534)。
Layer 1
Linux 命名空间
--unshare-user/pid/net/ipc/cgroup/uts：用户、PID、网络、IPC、cgroup、主机名隔离。--die-with-parent 父进程退出时沙箱一并退出。

7.1 Landlock 启动链（为何 /jiuwenbox 可不进白名单）

①
launcher 启动bwrap 挂载完成后，从 /jiuwenbox 预读 daemon 源码到内存
②
应用 Landlock锁定路径访问规则；此后无需再从磁盘读 daemon 脚本
③
compile + exec同 Python 进程变为 daemon（无二次 execve）→ 内核视角 launcher 与 daemon 为同一 PID 1
④
用户子进程fork+exec 继承 Landlock + seccomp；close_fds=True 不继承控制 listener fd

7.2 网络模式对比

mode行为
host不隔离网络，沙箱与宿主机共享网络栈
isolated独立 netns + veth；策略化 egress/ingress；管理面端口对沙箱不可达

8. 沙箱内 daemon 设计要点

控制 socket 在宿主机

box-server 在 spawn bwrap 之前 bind listener，fd 经 pass_fds 传入。沙箱内用户代码看不到、删不掉该 socket 路径。

一次 bwrap，多次 exec

namespace / 挂载 / seccomp / Landlock 在 daemon 启动时固化；每条用户命令仅 fork+exec，成本低。

防杀 daemon

seccomp 阻止 kill PID 1/2；每次用户 exec 独立 session，kill(0) 无法波及 daemon。

标准库-only

daemon 仅用标准库，可用 python3 -S 冷启动，launcher 的 compile/exec 不依赖外部包。

宿主机：bind(control.sock) → pass_fds → bwrap → launcher → daemon accept(fd)
沙箱内：用户 exec 不继承 listener fd · 共享 daemon 的 UID(nobody) · 继承隔离规则

11. 平台前提与运维

运行环境

  • Linux + bwrap
  • Landlock / seccomp / cgroup 按内核能力启用
  • Docker 常需 --privileged 或足够 capability

健康探测

GET /health 返回 landlock_supported: true 表示当前内核支持 Landlock 路径隔离。

运维注意

  • EDP 重启换新 sandbox → Adapter 宜重启或清缓存
  • from_logs 依赖 LOG_DIR 产出 process_*.log
  • 服务重启不恢复内存中的沙箱表
