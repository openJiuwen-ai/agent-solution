# 部署快速入门

本指南帮助开发人员快速在本地构建、配置、启动和部署 Adapter-Versatile-Agent 服务。

---

## 1. 前置条件

| 软件 | 版本要求 | 验证命令 |
|------|----------|----------|
| JDK | 17+ / 21（LTS 版本推荐） | `java -version` |
| Maven | 3.6+ | `mvn -version` |
| Versatile 服务 | 可访问（或使用 mock 服务） | `curl <versatile-url>` |

> 💡 **JDK 版本说明**：项目 pom.xml 中配置 `<java.version>17</java.version>`，使用 JDK 17 或 JDK 21 均可正常编译运行。

## 4. 最小配置

本服务遵循 Spring Boot 配置规范，支持通过**环境变量**、**JVM 参数**、**配置文件**多种方式配置。最小配置只需要设置 Versatile 服务地址。

### 4.1 通过环境变量配置（推荐）

**Windows PowerShell**：
```powershell
$env:VERSATILE_URL="http://localhost:30001/v1/0/agent-manager/workflows/mock_workflow/conversations/{conversation_id}?type=controller&workspace_id=10"
```

**Linux/macOS**：
```bash
export VERSATILE_URL="http://localhost:30001/v1/0/agent-manager/workflows/mock_workflow/conversations/{conversation_id}?type=controller&workspace_id=10"
```

### 4.2 使用 mock 服务

如果你暂时没有可用的 Versatile 服务，可以使用项目提供的 mock 服务 versatile_main.py（位于 agent-runtime 测试资源中），默认监听端口 30001。

### 4.3 配置示例说明

默认配置（定义于 [application.yml:18-36](file:///D:/AgentTool/EDPAgent_java/agent-solution/common/agent/adapter-versatile-agent-java/src/main/resources/application.yml#L18-L36)）：

```yaml
openjiuwen:
  service:
    versatile:
      url-template: ${VERSATILE_URL:http://localhost:30001/v1/0/agent-manager/workflows/mock_workflow/conversations/{conversation_id}?type=controller&workspace_id=10}
      timeout: ${VERSATILE_TIMEOUT:600s}
      forward-header-whitelist:
        - stream
        - x-invoke-mode
        - x-language
      headers-template:
        Content-Type: application/json
        accept: "*/*"
        stream: "true"
        x-invoke-mode: DEBUG
        x-language: zh-cn
      result-node-name: ${VERSATILE_RESULT_NODE:GXZQAResponseNode}
```

> ⚠️ **注意**：`{conversation_id}` 是占位符，会在每次请求时自动替换为实际的会话 ID。

## 6. 验证服务

服务启动后，通过以下步骤验证服务正常运行。

### 6.1 健康检查

```bash
curl http://localhost:8191/actuator/health
```

预期响应：
```json
{"status":"UP"}
```

### 6.2 获取 Agent Card

```bash
curl http://localhost:8191/.well-known/agent-card.json
```

预期返回标准 A2A Agent Card JSON，包含：
- `name`: "versatile-agent"
- `description`: 服务描述
- `skills`: 包含 `banking_workflow` 技能
- `url`: A2A 端点地址
- `capabilities`: 支持 streaming 等能力

### 6.3 发送测试 A2A 请求

使用 curl 发送一个测试的 A2A `sendStreamingMessage` 请求（SSE 流式）：

```bash
curl -N -X POST http://localhost:8191/a2a \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{
    "jsonrpc": "2.0",
    "id": "test-001",
    "method": "sendStreamingMessage",
    "params": {
      "message": {
        "messageId": "msg-001",
        "role": "user",
        "parts": [
          {
            "kind": "text",
            "text": "查询我的账户余额"
          }
        ]
      },
      "metadata": {
        "intent": "banking_workflow"
      }
    }
  }'
```

参数说明：
- `-N`：禁用 curl 缓冲，实时显示 SSE 流
- `Accept: text/event-stream`：要求 SSE 流式响应
- `metadata.intent`：可选，用于多意图路由

> 💡 **提示**：如果使用 mock 服务，你会看到 SSE 事件流逐帧返回 Versatile 的节点数据。

## 7. 核心配置说明（最小配置集）

所有配置项均可通过环境变量覆盖，格式为：将 YAML 配置名转为大写，用下划线连接。

| 环境变量 | YAML 配置路径 | 默认值 | 说明 |
|----------|---------------|--------|------|
| `VERSATILE_URL` | `openjiuwen.service.versatile.url-template` | `http://localhost:30001/...` | **必填**：Versatile 工作流 URL 模板，`{conversation_id}` 会自动替换 |
| `VERSATILE_RESULT_NODE` | `openjiuwen.service.versatile.result-node-name` | `GXZQAResponseNode` | 结果节点名称，匹配该名称的节点将返回给 LLM |
| `VERSATILE_TIMEOUT` | `openjiuwen.service.versatile.timeout` | `600s` | Versatile 请求超时时间，单位支持 s/m/h |
| `VERSATILE_AGENT_PORT` | `server.port` | `8191` | 服务监听端口 |
| `VERSATILE_AGENT_TENANT_ID` | `agent-runtime.access.a2a.default-tenant-id` | `edp-tenant` | A2A 默认租户 ID |

### 7.1 URL 模板说明

URL 模板支持的占位符：
- `{conversation_id}`：自动替换为 A2A 会话 ID（必填）
- `{workflow_id}`：工作流 ID（可配置默认值）

URL 中的 query 参数可直接写在模板中，如 `?type=controller&workspace_id=10`。

### 7.2 Header 透传配置

默认透传白名单（[application.yml:31-33](file:///D:/AgentTool/EDPAgent_java/agent-solution/common/agent/adapter-versatile-agent-java/src/main/resources/application.yml#L31-L33)）：
- `x-invoke-mode`：调用模式（DEBUG 等）
- `x-language`：语言（zh-cn、en-us 等）

需要透传其他 Header 时，在 `passthrough-headers` 列表中添加。

## 9. 与 EDPAgent 联调

当本服务在本地启动后，可以配置 EDPAgent 连接到本服务进行联调。

### 9.1 EDPAgent 配置

在 EDPAgent 的配置中设置：

```properties
# EDPAgent application.properties / application.yml
EDP_AGENT_VERSATILE_A2A_URL=http://localhost:8191/a2a
```

或通过环境变量：

**Windows PowerShell**：
```powershell
$env:EDP_AGENT_VERSATILE_A2A_URL="http://localhost:8191/a2a"
```

**Linux/macOS**：
```bash
export EDP_AGENT_VERSATILE_A2A_URL="http://localhost:8191/a2a"
```

### 9.2 联调步骤

1. 启动 Versatile mock 服务（端口 30001）或真实 Versatile 服务
2. 启动本服务（端口 8191）
3. 启动 EDPAgent（配置 `EDP_AGENT_VERSATILE_A2A_URL` 指向本服务）
4. 在 EDPAgent 对话中触发银行业务相关请求，观察：
   - EDPAgent 日志中是否调用了 versatile-agent
   - 本服务日志中是否收到 A2A 请求并转发给 Versatile
   - 前端是否收到菜单/表单等透传消息
   - 最终结果是否正确返回给 EDPAgent

### 9.3 常见联调问题排查

| 问题 | 排查方向 |
|------|----------|
| 连接被拒绝 | 检查本服务是否正常启动，端口 8191 是否监听 |
| Agent Card 获取失败 | 检查访问路径是否为 `/.well-known/agent-card.json` |
| Versatile 请求超时 | 检查 `VERSATILE_URL` 是否可达，网络是否连通，`VERSATILE_TIMEOUT` 是否足够 |
| 没有返回结果节点 | 检查结果节点名配置是否与实际 Versatile 返回的 `node_name` 一致 |
| Header 没有透传 | 检查 Header 名是否在 `passthrough-headers` 白名单中 |
