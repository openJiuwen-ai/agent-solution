# 产品介绍

## EDPAgent 是什么

EDPAgent（Enterprise Dynamic Planning Agent）是一款面向企业场景的**通用动态规划智能体**。它基于 Openjiuwen DeepAgent（Reasoning + Acting）循环范式，能够自主进行任务规划、工具调用、结果观察与动态路径调整，最终完成复杂的业务目标。

EDPAgent 采用"配置驱动"的设计理念——**角色定义、行为边界、工具约束、业务话术、技能路由**全部通过 YAML 配置声明，无需修改 Java 代码即可适配不同行业场景。

## 核心能力

### 动态任务规划

基于Openjiuwen DeepAgent循环自动拆解任务、创建 Todo 列表、管理任务状态机（PENDING → IN_PROGRESS → COMPLETED/CANCELLED），支持动态路径调整和依赖闭环检测。

- 自动规划：根据用户意图创建任务清单
- 状态管理：严格的状态机流转，所有状态变更通过工具调用完成
- 动态调整：执行过程中可根据中间结果跳过或取消任务
- 循环重规划：支持多轮对话，完成后可根据新意图重新规划

### Skill 技能系统

通过声明式 YAML 定义技能触发词、业务话术和 Python 执行脚本，支持三种加载模式（all/auto_list/none）。

- 零代码扩展：YAML 声明元数据 + Python 编写执行逻辑
- 触发词路由：通过 description 中的触发词描述实现自动匹配
- 话术模板：内置多场景话术，支持变量替换
- 热插拔：放入目录重启即可生效

### 场景化配置

通过 `EDP_AGENT_SCENARIO_HOME` 环境变量一键切换业务场景，场景级 Governance 配置增量覆盖框架默认配置。

- 多场景并存：同一套代码支持多个业务场景
- 增量覆盖：场景配置仅需写差异部分，继承框架默认值
- 隔离性：场景间配置完全隔离，切换无残留

### 外部工作流委托

通过 `call_versatile` 工具委托外部 Versatile 服务执行业务工作流，支持 REST 直连和 A2A 适配器两种模式，支持中断/续传。

- 两种调用模式：REST API 直连、A2A 协议适配
- 中断续传：支持用户交互中断，恢复后从断点继续
- 结果归一化：通过脚本对 Versatile 返回结果进行标准化处理

### MCP 脚本沙箱

通过 `call_mcp` 工具调用 MCP（Model Context Protocol）SSE 服务执行脚本，支持主备双 URL 切换。

- 沙箱执行：Python 脚本在 MCP 沙箱中运行，安全隔离
- 主备切换：配置主备 URL，自动故障转移
- Token 鉴权：支持 Access Token 认证

### A2A 协议通信

对外暴露标准 A2A（Agent-to-Agent）JSON-RPC over SSE 接口，可与任何 A2A 兼容的 Agent 网络互通。

- 标准协议：遵循 Google A2A 协议规范
- SSE 流式：服务端推送事件，实时返回执行进度
- Agent Card 发现：通过 `/.well-known/agent-card.json` 自动发现

### 内置工具体系

提供5个核心业务工具：

- `call_versatile`：委托 Versatile 执行工作流
- `call_mcp`：调用 MCP 沙箱执行脚本
- `ask_user`：向用户追问缺失信息（触发中断）
- `cancel_task`：取消当前任务
- `todo_create/todo_modify/todo_list/todo_get`：任务状态管理

### 工具间数据直通

通过 ToolDataChannel 机制实现工具间数据传递，前序工具输出可直接作为后序工具输入，无需依赖 LLM 转述。

## Rail 拦截器链

EDPAgent 通过 9 个 Rail 拦截器构成处理链，按顺序执行：

| 顺序 | Rail 名称 | 职责 |
|------|-----------|------|
| 1 | CancelRail | 检测任务取消请求，触发取消流程 |
| 2 | EdpaTodoRail | 处理 Todo 工具调用，维护 Redis 中的任务状态 |
| 3 | ExecutionLimitRail | 检查执行步数和子任务数限制，防止死循环 |
| 4 | McpInterruptRail | 处理 call_mcp 意图，调用 MCP SSE 执行脚本 |
| 5 | VersatileInterruptRail | 处理 call_versatile 意图，调用 Versatile 工作流 |
| 6 | AskUserTemplateRail | 处理 ask_user 调用，注入话术模板触发中断 |
| 7 | LogRail | 记录关键操作日志 |
| 8 | EdpaEventRail | 发送 SSE 事件流，推送状态更新 |
| 9 | ScriptsRail | 处理话术渲染，返回标准化脚本 |

## 适用场景

EDPAgent 适用于需要"智能规划+工具调用+人机交互"的企业业务场景：

- **金融理财**：产品推荐、购买流程、资金筹划
- **客服助手**：多轮对话、业务办理、问题解答
- **智能运维**：故障诊断、操作执行、报告生成
- **政务办理**：材料审核、流程引导、多部门协同
- **通用 Agent 平台**：作为 Agent 运行时承载各类业务场景
