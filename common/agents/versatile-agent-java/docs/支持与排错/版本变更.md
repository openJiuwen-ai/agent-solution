# 版本变更记录

本文档记录 adapter-versatile-agent-java 各版本的变更内容。

## v0.1.0-SNAPSHOT（当前版本）

**发布日期**：2026年7月

### 新特性

- ✅ **独立A2A代理进程部署**：
  - 作为独立Spring Boot应用部署，与EDPAgent主进程隔离
  - 轻量级无状态设计，无Redis、无数据库、无MCP、无Python依赖
  - 纯Java实现，仅需JRE即可运行
  - 默认端口8191，可通过`VERSATILE_AGENT_PORT`配置

- ✅ **Versatile REST/SSE协议适配**：
  - 封装HttpClient，处理Versatile REST API调用
  - 支持SSE（Server-Sent Events）流式响应解析
  - 内置VersatileRequestExtractor和VersatileResponseExtractor
  - 核心组件`VersatileAgentHandler`一键注册为Spring Bean

- ✅ **节点类型三分流机制**：
  - **Target.USER透传**：非结果节点（菜单、提示、中间过程）直接透传前端
  - **Target.LLM结果**：匹配resultNodeName的结果节点回灌LLM处理
  - **Interrupt中断**：缺少结果节点或需要用户输入时触发A2A中断
  - 保留节点结构化信息（菜单按钮、卡片格式等）不被LLM降维

- ✅ **URL模板占位符替换**：
  - URL模板支持`{conversation_id}`占位符
  - 运行时自动替换为A2A请求中的contextId
  - 支持自定义查询参数（如`type=controller&workspace_id=10`）
  - 通过`VERSATILE_URL`环境变量配置

- ✅ **Header白名单透传机制**：
  - 默认白名单：`stream`、`x-invoke-mode`、`x-language`
  - 白名单外的Header默认不透传，避免Header泄露
  - 支持配置`forward-header-whitelist`自定义白名单
  - 默认Headers自动注入：Content-Type、accept、stream、x-invoke-mode、x-language

- ✅ **多意图endpoints路由支持**：
  - 通过`input-metadata-keys`提取A2A metadata中的intent字段
  - intent自动注入Versatile请求body，支持Versatile侧路由
  - 默认提取字段：`intent`、`wap_userName`

- ✅ **结果节点名可配置**：
  - 通过`VERSATILE_RESULT_NODE`环境变量配置（默认`GXZQAResponseNode`）
  - 适配不同Versatile工作流的结果节点命名
  - 支持按工作流类型配置不同的结果节点名（需要扩展）

- ✅ **Docker容器化部署**：
  - 基于eclipse-temurin:21-jre-jammy基础镜像
  - 内置curl用于HEALTHCHECK
  - 默认EXPOSE 8191端口
  - Docker HEALTHCHECK通过`/.well-known/agent-card.json`
  - HEALTHCHECK配置：interval=10s, timeout=5s, retries=8, start-period=90s

- ✅ **Spring Boot Actuator健康检查**：
  - 暴露`/actuator/health`健康检查端点
  - 暴露`/actuator/info`信息端点
  - 与Docker HEALTHCHECK集成
  - 启动时不依赖Versatile后端连通性（Versatile不可达不影响启动）

- ✅ **标准A2A Agent Card发现**：
  - 端点`/.well-known/agent-card.json`
  - 返回agentId、description、skills等元数据
  - 符合A2A协议标准服务发现机制
  - 同时用于Docker健康检查

- ✅ **可配置超时机制**：
  - 默认超时600秒（10分钟），适配长工作流
  - 通过`VERSATILE_TIMEOUT`环境变量配置
  - 支持Duration格式：ms（毫秒）、s（秒）、m（分钟）

- ✅ **DEBUG日志支持**：
  - 默认开启`com.huawei.ascend.versatile`和`com.openjiuwen.service.adapters.versatile`包DEBUG日志
  - 日志文件：`logs/run/run.log`
  - 记录完整请求URL、Headers、响应、SSE节点内容，便于排错

### 技术栈

- Spring Boot 4.0.6
- Java 17（编译）/ JDK 21（Docker运行时）
- agent-service-app 0.1.0（A2A运行时）
- agent-service-adapters-versatile 0.1.0-SNAPSHOT（Versatile适配核心）
- SLF4J + Logback 日志
- Maven 构建

### 核心配置项

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `VERSATILE_AGENT_PORT` | `8191` | 服务监听端口 |
| `VERSATILE_URL` | `http://localhost:30001/v1/0/agent-manager/workflows/mock_workflow/conversations/{conversation_id}?type=controller&workspace_id=10` | Versatile URL模板 |
| `VERSATILE_TIMEOUT` | `600s` | Versatile调用超时 |
| `VERSATILE_RESULT_NODE` | `GXZQAResponseNode` | 结果节点名称 |

### 端点列表

| 端点 | 方法 | 说明 |
|------|------|------|
| `/a2a` | POST | A2A协议端点（SendStreamingMessage） |
| `/.well-known/agent-card.json` | GET | Agent Card发现/健康检查 |
| `/actuator/health` | GET | Actuator健康检查 |
| `/actuator/info` | GET | Actuator应用信息 |

## 升级指南

### 从历史版本升级

当前为初始开发版本（v0.1.0-SNAPSHOT），暂无历史版本升级路径。后续版本升级指南将在此处提供。

### 升级注意事项

1. **备份配置**：升级前记录当前环境变量配置（特别是VERSATILE_URL、VERSATILE_RESULT_NODE等）
2. **无状态升级**：由于是无状态服务，直接替换镜像/重启即可，不需要数据迁移
3. **先测试验证**：在测试环境验证Versatile调用正常、节点分流正确后再升级生产
4. **查看changelog**：检查是否有配置项变更或废弃
5. **日志级别调整**：生产环境建议将日志级别调整为INFO，避免DEBUG日志过多
