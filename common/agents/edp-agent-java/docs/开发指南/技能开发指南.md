# Skill 开发指南

## 什么是 Skill

Skill 是 EDPAgent 的技能扩展单元。每个 Skill 封装了一个特定的业务能力（如"理财产品推荐"、"余额查询"、"产品选择确认"等），通过声明式 YAML 定义元数据和触发规则，通过 Python 脚本实现业务逻辑。

**Skill 的核心特点**：
- **零 Java 代码**：YAML 声明 + Python 脚本，无需修改引擎代码
- **触发词路由**：通过 description 中的触发词描述，LLM 自动判断何时使用哪个 Skill
- **话术模板**：内置多场景话术，支持变量替换
- **热插拔**：放入目录重启即可生效

## Skill 目录结构

每个 Skill 是场景目录 `skills/` 下的一个子目录，结构如下：

```
scenarios/<场景名>/skills/<skill_name>/
├── SKILL.yaml          # Skill 元数据（必填）
├── SKILL.md            # Skill 使用说明（给 Agent 参考，可选但推荐）
└── scripts/            # Python 脚本目录（可选）
    ├── run_xxx.py      # 执行脚本
    └── ...             # 其他辅助脚本/依赖
```

以 wealth-demo 场景的 4 个 Skill 为例：

```
scenarios/wealth-demo/skills/
├── product_recommend_skill/        # 产品推荐
│   ├── SKILL.yaml
│   ├── SKILL.md
│   └── scripts/
│       └── run_product_recommend_skill.py
├── product_select_skill/           # 产品选择
│   ├── SKILL.yaml
│   └── SKILL.md                    # 无脚本，纯话术
├── fund_planning_skill/            # 资金筹划
│   ├── SKILL.yaml
│   ├── SKILL.md
│   └── scripts/
│       └── run_fund_planning.py
└── interact_finance_rec_skill/     # 交互式推荐（换一批/筛选）
    ├── SKILL.yaml
    ├── SKILL.md
    └── scripts/
        ├── run_interact_finance_rec_skill.py
        ├── run_mcp_recommend.py
        ├── run_versatile_normalize.py
        ├── mcp_sse_client.py
        └── log_utils.py
```

## SKILL.yaml 详解

SKILL.yaml 是 Skill 的核心定义文件，包含以下字段：

### 完整字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | Skill 唯一标识名，使用小写字母+下划线命名，与目录名一致 |
| `description` | string | 是 | Skill 功能描述 + 触发词 + 禁止场景。**此描述直接作为 System Prompt 的一部分发给 LLM**，决定了 Skill 何时被触发 |
| `scripts` | map | 否 | 话术模板 key-value 对，key 为话术名，value 为话术文本 |

### description 编写规范

description 是 Skill 开发中**最重要**的字段，直接影响 LLM 是否能正确触发 Skill。

**推荐结构**（三段式）：
1. **功能说明**：第一句说明这个 Skill 做什么
2. **触发词列举**：列出典型的用户触发词
3. **排除场景**：明确说明不要用于什么场景

**示例**（product_recommend_skill）：

```yaml
description: "根据用户理财意图进入理财推荐流程并展示推荐产品列表。触发词：推荐理财产品、帮我看看理财、有什么理财可以买、推荐稳健理财。不要用于：产品选择确认、资金筹划、账户查询、基金购买、股票交易。"
```

**最佳实践**：
- ✅ 使用中文编写，与用户输入语言一致
- ✅ 触发词列举要具体，包含用户的各种说法（同义词、口语化表达）
- ✅ 排除场景要明确，减少误触发
- ✅ 描述简洁准确，控制在 200 字以内
- ❌ 不要写得太长，否则会稀释 System Prompt
- ❌ 不要只写功能不写触发词
- ❌ 不要使用模糊的描述如"处理理财相关请求"

### scripts 话术模板

scripts 定义 Skill 使用的话术模板，Agent 执行 Skill 时可通过 key 引用对应话术返回给用户。

**示例**：

```yaml
scripts:
  product_recommend_success: "我找到以上您可能感兴趣的产品，可以告诉我购买哪支产品及购买金额。"
  product_recommend_empty: '我理解你对稳健收益的追求，但"稳赚不赔"的产品在金融领域并不存在。'
  mcp_result_empty: "根据您的条件没有找到合适产品，您可以从以下产品中选择一个或者重新筛选。"
  amount_prompt: "请告诉我您想购买的金额（最低1万元，以万元为单位递增）。"
```

**话术使用场景**：
- 工具调用完成后返回成功/失败消息
- ask_user 追问时的提示语
- 异常情况的友好提示
- 流程引导语

## SKILL.md 编写

SKILL.md 是给 LLM Agent 阅读的 Skill 使用说明，相当于 Skill 的"帮助文档"。当 Skill 逻辑较复杂时（如涉及多步工具调用、条件分支），应在 SKILL.md 中详细说明使用方法。

**建议包含的内容**：
1. Skill 的功能和适用场景
2. 前置条件（需要什么信息）
3. 执行步骤（应该调用哪些工具，按什么顺序）
4. 工具参数说明
5. 异常处理
6. 注意事项

**示例**（product_recommend_skill/SKILL.md 结构）：

```markdown
# 产品推荐 Skill

## 功能
根据用户的理财需求，推荐合适的理财产品。

## 触发时机
- 用户首次请求推荐理财产品
- 用户说"帮我看看理财"、"有什么理财可以买"

## 执行流程
1. 如果用户未提供风险偏好和金额，先 ask_user 追问
2. 调用 call_mcp 执行推荐脚本（script_command: recommend_products）
3. 读取 MCP 返回结果
4. 如果推荐结果为空，使用 mcp_result_empty 话术
5. 展示推荐结果后，使用 product_recommend_success 话术引导用户选择
6. 调用 todo_modify 将当前任务标记完成，等待用户选择产品

## 注意事项
- 不要推荐 scope.denied 中禁止的产品（基金、股票、保险等）
- 推荐结果不超过5个产品
- ...
```

## Python 脚本开发

当 Skill 需要调用外部数据、执行计算或调用业务接口时，使用 Python 脚本。

### 脚本规范

1. **入口函数**：脚本应有一个主入口函数（如 `main`、`run_xxx`）
2. **参数接收**：通过函数参数或命令行参数接收 `script_params`
3. **返回格式**：必须返回 JSON 可序列化的 dict，结果会自动写入 ToolDataChannel
4. **异常处理**：内部捕获异常，返回有意义的错误信息
5. **无副作用**：查询类脚本只读不写；有状态变更的操作通过 Versatile 执行

### 脚本示例：产品推荐

```python
# scripts/run_product_recommend_skill.py
import json
import sys

def run_product_recommend(params):
    """
    理财产品推荐入口
    :param params: dict, 包含 risk_level, amount 等参数
    :return: dict, 推荐结果
    """
    risk_level = params.get("risk_level", "R2")
    amount = params.get("amount", 0)
    
    # 1. 参数校验
    if amount < 10000:
        return {
            "success": False,
            "error": "最低购买金额为1万元",
            "error_script_key": "amount_too_small"
        }
    
    # 2. 调用业务接口或MCP获取数据
    # 这里是Mock，实际应 call_mcp 或调用业务API
    products = [
        {
            "id": "P001",
            "name": "稳健理财A",
            "rate": "3.2%",
            "term": "90天",
            "risk": "R2",
            "min_amount": 10000
        },
        {
            "id": "P002", 
            "name": "增值理财B",
            "rate": "4.1%",
            "term": "180天",
            "risk": "R3",
            "min_amount": 10000
        }
    ]
    
    # 3. 按风险等级过滤
    filtered = [p for p in products if p["risk"] <= risk_level]
    
    # 4. 返回结构化结果
    return {
        "success": True,
        "products": filtered,
        "total_count": len(filtered),
        "recommend_reason": f"根据您的风险偏好{risk_level}，为您推荐{len(filtered)}款产品"
    }

if __name__ == "__main__":
    # 支持命令行直接测试
    if len(sys.argv) > 1:
        params = json.loads(sys.argv[1])
    else:
        params = {}
    
    result = run_product_recommend(params)
    print(json.dumps(result, ensure_ascii=False))
```

### 在 Agent 中调用脚本

脚本通过 `call_mcp` 工具调用：

```json
{
  "name": "call_mcp",
  "arguments": {
    "script_command": "run_product_recommend_skill",
    "script_params": {
      "risk_level": "R2",
      "amount": 50000
    }
  }
}
```

执行结果自动写入 ToolDataChannel，后续可通过 `input_key` 读取。

### 脚本调试

本地调试脚本：

```bash
cd scenarios/wealth-demo/skills/product_recommend_skill
python scripts/run_product_recommend_skill.py '{"risk_level":"R2","amount":50000}'
```

## 理财场景4个Skill样例解析

### 1. product_recommend_skill（产品推荐）

**SKILL.yaml**：
```yaml
name: product_recommend_skill
description: "根据用户理财意图进入理财推荐流程并展示推荐产品列表。触发词：推荐理财产品、帮我看看理财、有什么理财可以买。不要用于：产品选择确认、资金筹划、账户查询。"
scripts:
  product_recommend_success: "我找到以上您可能感兴趣的产品，可以告诉我购买哪支产品及购买金额。"
  product_recommend_empty: '我理解你对稳健收益的追求，但"稳赚不赔"的产品在金融领域并不存在。'
  mcp_result_empty: "根据您的条件没有找到合适产品，您可以从以下产品中选择一个或者重新筛选。"
```

**触发时机**：用户首次表达理财需求
**执行模式**：调用 call_mcp 获取推荐列表 → 展示 → 引导用户选择

### 2. product_select_skill（产品选择）

**SKILL.yaml**：
```yaml
name: product_select_skill
description: "当用户从推荐列表中选择了具体产品后触发，处理产品选择确认。触发词：我买第一个、就选这个、买P001、买稳健理财A。不要用于：首次推荐请求、资金筹划。"
scripts:
  confirm_purchase: "您选择了{product_name}，预期年化收益率{rate}，期限{term}。请问确认购买吗？"
```

**触发时机**：用户从推荐结果中选择具体产品
**执行模式**：纯话术确认 → ask_user 确认购买 → 调用 call_versatile 执行购买

### 3. fund_planning_skill（资金筹划）

**触发时机**：用户确认购买但余额不足，或需要在理财卡和储蓄卡之间调配资金
**执行模式**：查询余额 → 计算缺口 → 提示转账 → 执行转账后回到购买流程

### 4. interact_finance_rec_skill（交互式推荐）

**触发时机**：用户说"换一批"、"有没有短期的"、"风险低一点的"等追加筛选条件
**执行模式**：解析新条件 → 调用 MCP 重新查询 → 返回新推荐结果

## Skill 加载与配置

### 加载模式

在 actrule.yaml 中配置 `skill_mode`：

```yaml
actrule:
  skill_mode: all  # all / auto_list / none
```

| 值 | 说明 |
|----|------|
| `all` | 启动时加载所有 Skill 的 description 到 System Prompt，Agent 直接判断匹配（默认，推荐） |
| `auto_list` | Agent 先调用 list_tool 查询匹配 Skill，适合 Skill 数量很多（>20个）的场景 |
| `none` | 不加载任何 Skill |

### 启用工具

确保 actrule.yaml 的 allowed_tools 包含 `skill_tool`（框架默认已包含）：

```yaml
actrule:
  allowed_tools:
    - skill_tool
    - ...
```

### 加载新 Skill

1. 将 Skill 目录放入场景的 `skills/` 目录
2. 重启服务（当前版本不支持热加载）
3. 查看启动日志，确认 Skill 被加载：
   ```
   Loaded skill: product_recommend_skill
   Loaded skill: product_select_skill
   ```

## Skill 路由配置

在场景的 planrule.yaml 中配置 `skill_routing`，显式声明触发条件与 Skill 的映射：

```yaml
planrule:
  skill_routing:
    - trigger: "用户首次请求推荐理财产品"
      skill: "product_recommend_skill"
      priority: 1
    - trigger: "用户从推荐结果中选择产品"
      skill: "product_select_skill"
      priority: 2
    - trigger: "用户确认购买或需要资金筹划"
      skill: "fund_planning_skill"
      priority: 2
    - trigger: "用户请求换一批、追加筛选条件"
      skill: "interact_finance_rec_skill"
      priority: 1
```

priority 越小优先级越高。当有多个 Skill 可能匹配时，优先选择 priority 小的。

## 注意事项

1. **命名规范**：Skill 目录名、SKILL.yaml 中的 name 字段必须一致，使用小写字母+下划线。

2. **description 是核心**：花时间打磨 description，这是 Skill 能否被正确触发的关键。写完后要做触发测试。

3. **SKILL.md 要写清楚**：复杂的 Skill 一定要写 SKILL.md，告诉 LLM 如何正确使用这个 Skill（先做什么、后做什么、调用什么工具、异常怎么处理）。

4. **脚本要幂等**：Python 脚本尽量做到幂等，网络重试不会造成重复操作。有副作用的操作（如扣款）通过 Versatile 执行，不要直接在 MCP 脚本中调用。

5. **错误处理友好**：脚本出错时返回友好的错误消息（通过 error_script_key 引用话术），不要把技术堆栈暴露给用户。

6. **不要硬编码**：API 地址、密钥等配置不要硬编码在脚本中，通过环境变量或配置文件传入。

7. **测试验证**：每个 Skill 开发完成后，必须通过发送触发词测试是否能正确触发，并验证完整流程。

## 常见问题

### Skill 不被触发

可能原因：
- description 中没有包含用户使用的触发词
- description 写得太长或太模糊，LLM 无法判断
- skill_mode 设置为 none
- 目录名与 SKILL.yaml 的 name 不一致
- 检查日志中是否有 Skill 加载错误

### Skill 误触发（不该触发时触发了）

可能原因：
- description 写得太宽泛，没有明确排除场景
- 没有写"不要用于"的排除说明
- priority 配置不合理

### 脚本执行失败

排查步骤：
1. 查看 Agent 日志中的 Python 错误堆栈
2. 本地直接运行脚本测试
3. 确认 MCP 服务可用且已配置
4. 检查 script_params 参数是否正确传递
