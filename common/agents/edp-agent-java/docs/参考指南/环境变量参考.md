## 服务端口

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `SERVER_PORT` | `8190` | 否 | HTTP 服务监听端口。Docker 默认暴露 8190。 |

**示例**：
```bash
SERVER_PORT=8080
```

## Redis 连接

### 基础连接（Spring Data Redis）

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDPA_REDIS_HOST` | `localhost` | *部署时必填 | Redis 主机地址。Docker 连宿主机用 `host.docker.internal`（Win/Mac）或宿主机IP（Linux）。 |
| `EDPA_REDIS_PORT` | `6379` | 否 | Redis 服务端口。 |
| `EDPA_REDIS_PASSWORD` | （空） | 否 | Redis 密码，无密码留空。 |
| `EDPA_REDIS_DB` | `0` | 否 | Redis 数据库编号（0-15）。 |

### Redis 模式与高级配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDPA_REDIS_MODE` | `single` | 否 | Redis 部署模式：<br>- `single`：单机模式<br>- `sentinel`：哨兵模式<br>- `cluster`：集群模式 |
| `EDPA_REDIS_CONNECT_TIMEOUT` | `5000` | 否 | 连接超时时间，单位毫秒。 |
| `EDPA_REDIS_SOCKET_TIMEOUT` | `10000` | 否 | Socket 读写超时时间，单位毫秒。跨网络部署建议增大到 20000。 |
| `EDPA_REDIS_CHECKPOINTER_TTL` | `60` | 否 | Checkpoint（中断恢复点）数据 TTL，单位分钟。默认60分钟。需要支持长中断续传可增大（如1440=24小时）。 |
| `EDPA_REDIS_KEY_PREFIX` | `edpa` | 否 | Redis Key 前缀，多实例共用同一 Redis 时用于区分。 |
| `EDPA_REDIS_TODO_TTL` | `3600` | 否 | Todo 任务数据 TTL，单位秒。默认3600秒（1小时）。 |
| `EDPA_REDIS_REFRESH_ON_READ` | `true` | 否 | 读取 Todo 数据时是否自动续期 TTL。`true`=活跃会话不过期。 |

**生产环境示例**：
```bash
EDPA_REDIS_MODE=single
EDPA_REDIS_HOST=redis.internal.example.com
EDPA_REDIS_PORT=6379
EDPA_REDIS_PASSWORD=your-secure-password
EDPA_REDIS_DB=0
EDPA_REDIS_CONNECT_TIMEOUT=5000
EDPA_REDIS_SOCKET_TIMEOUT=10000
EDPA_REDIS_CHECKPOINTER_TTL=1440
EDPA_REDIS_KEY_PREFIX=edpa-prod
EDPA_REDIS_TODO_TTL=7200
EDPA_REDIS_REFRESH_ON_READ=true
```

## Agent 核心配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDPA_AGENT_CONFIG_PATH` | `src/main/resources/edp-config.yaml` | 否 | 配置锚点文件路径。Docker 镜像中已设置为 `/app/config/edp-config.yaml`，一般不需要修改。 |
| `EDP_AGENT_SCENARIO_HOME` | `../scenarios/wealth-demo` | **是** | **场景目录路径**。切换此值切换业务场景。Docker 中默认 `/app/scenarios/wealth-demo`。自定义场景需挂载并指向对应路径。 |

**示例**：
```bash
# 使用理财演示场景（Docker 默认）
EDP_AGENT_SCENARIO_HOME=/app/scenarios/wealth-demo

# 使用智贷通场景
EDP_AGENT_SCENARIO_HOME=/app/scenarios/hz-zhidaitong

# 使用自定义场景（挂载到容器内）
EDP_AGENT_SCENARIO_HOME=/app/scenarios/my-business-scenario
```

## 大模型配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_AGENT_MODEL_PROVIDER` | `OpenAI` | 否 | 模型提供者。当前版本仅支持 OpenAI 兼容接口，填 `OpenAI`。 |
| `EDP_AGENT_MODEL_NAME` | `deepseek-v4-pro` | 否 | 模型名称。根据实际使用的模型填写（如 `gpt-4o`、`deepseek-chat` 等）。 |
| `EDP_AGENT_MODEL_BASE_URL` | `https://api.deepseek.com/v1` | 否 | 模型 API Base URL。私有部署时修改为你的模型服务地址。 |
| `EDP_AGENT_MODEL_API_KEY` | `sk-067b754046374d83888c52657ab6bbc7` | 否 | 大模型 API Key。<br>⚠️ **生产环境必须通过环境变量覆盖此默认值**，不要在生产中使用内置测试Key。 |

**DeepSeek（默认）**：
```bash
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=deepseek-v4-pro
EDP_AGENT_MODEL_BASE_URL=https://api.deepseek.com/v1
EDP_AGENT_MODEL_API_KEY=sk-your-deepseek-key
```

**OpenAI GPT**：
```bash
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=gpt-4o
EDP_AGENT_MODEL_BASE_URL=https://api.openai.com/v1
EDP_AGENT_MODEL_API_KEY=sk-your-openai-key
```

**私有部署模型（vLLM/Ollama等OpenAI兼容服务）**：
```bash
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=your-model-name
EDP_AGENT_MODEL_BASE_URL=http://your-llm-server:8000/v1
EDP_AGENT_MODEL_API_KEY=your-key
```

## Versatile 配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_AGENT_VERSATILE_URL` | `http://localhost:30001/v1/0/agent-manager/workflows/{workflow_id}/conversations/{conversation_id}` | 否 | Versatile REST API 地址模板。URL 中 `{workflow_id}` 和 `{conversation_id}` 为路径变量，运行时自动填充。不使用 Versatile 可保持默认（但 actrule 不要启用 call_versatile）。 |
| `EDP_AGENT_VERSATILE_A2A_URL` | `http://localhost:8191/a2a` | 否 | Versatile A2A 适配器地址。使用 A2A 模式对接 Versatile 时配置。 |
| `EDP_AGENT_VERSATILE_TIMEOUT` | `30s` | 否 | Versatile 调用超时时间。支持 Spring Duration 格式：`30s`(秒)、`1m`(分钟)、`60s`。生产环境建议 60s。 |

> **注意**：Versatile 的 `url-variables`（如 workflow_id）、`query-params`（type、workspace_id）、固定 `headers` 当前在 application.yml 中硬编码，暂不支持通过环境变量覆盖。如需修改这些值，需要修改 application.yml 或通过自定义 Spring Profile 覆盖。

**生产环境示例**：
```bash
EDP_AGENT_VERSATILE_URL=http://versatile.internal.example.com/v1/0/agent-manager/workflows/{workflow_id}/conversations/{conversation_id}
EDP_AGENT_VERSATILE_A2A_URL=http://versatile-a2a.internal.example.com/a2a
EDP_AGENT_VERSATILE_TIMEOUT=60s
```

## MCP SSE 配置

不使用 `call_mcp` 工具时可全部留空。

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_MCP_MASTER_URL` | （空） | 否 | MCP SSE 主服务 URL。格式：`http://host:port/sse`。 |
| `EDP_MCP_STANDBY_URL` | （空） | 否 | MCP SSE 备服务 URL。主节点不可用时自动切换到备节点。 |
| `EDP_MCP_ACCESS_TOKEN` | （空） | 否 | MCP SSE 鉴权 Token。需要认证时配置。 |
| `EDP_MCP_APP_NAME` | （空） | 否 | MCP 应用名称标识，用于 MCP 服务端识别调用方。 |

**主备模式示例**：
```bash
EDP_MCP_MASTER_URL=http://mcp-master:8080/sse
EDP_MCP_STANDBY_URL=http://mcp-standby:8080/sse
EDP_MCP_ACCESS_TOKEN=your-mcp-token
EDP_MCP_APP_NAME=edp-agent
```

**单节点示例**：
```bash
EDP_MCP_MASTER_URL=http://mcp-server:8080/sse
```

## 日志配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_AGENT_LOG_LEVEL` | `INFO` | 否 | ⚠️ **注意**：当前版本 `application.yml` 中日志级别为硬编码（默认包级别 DEBUG），并未直接通过此环境变量绑定。此变量在 `.env.example` 中定义，可通过 Spring Boot 启动参数 `-Dlogging.level.com.huawei.ascend.edp=${EDP_AGENT_LOG_LEVEL:-INFO}` 生效，或通过 Spring Profile 切换：<br>- `--spring.profiles.active=prod`：生产环境，日志级别 WARN<br>- `--spring.profiles.active=dev`：开发环境，日志级别 DEBUG<br>- 不指定 profile：使用 application.yml 默认配置（DEBUG 级别） |

**通过启动参数设置日志级别**：
```bash
java -Dlogging.level.com.huawei.ascend.edp=INFO \
     -Dlogging.level.com.openjiuwen=WARN \
     -jar edp-agent-engine.jar
```

**Docker 通过环境变量传递 JVM 参数**：
```bash
docker run -e JAVA_OPTS="-Dlogging.level.com.huawei.ascend.edp=INFO" ...
```

**使用 Spring Profile（推荐）**：
```bash
# 生产环境
docker run -e SPRING_PROFILES_ACTIVE=prod ...
```

## Python 配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `PYTHONIOENCODING` | `utf-8` | 否 | Python 标准输出编码。Dockerfile 中已设置为 `utf-8`，防止中文乱码。一般不需要修改。 |

## Dockerfile 预置环境变量

Docker 镜像中通过 ENV 指令预置了以下变量，使用 Docker 部署时无需重复设置：

| 环境变量 | Docker 中值 |
|----------|-------------|
| `SERVER_PORT` | `8190` |
| `EDPA_AGENT_CONFIG_PATH` | `/app/config/edp-config.yaml` |
| `EDP_AGENT_SCENARIO_HOME` | `/app/scenarios/wealth-demo` |
| `PYTHONIOENCODING` | `utf-8` |

如需覆盖，在 `docker run` 时通过 `-e` 传入即可。

```bash
EDP_AGENT_MODEL_NAME=deepseek-v4-pro

# ============================================
# MCP SSE 配置
# ============================================
EDP_MCP_MASTER_URL=http://mcp.example.com:8080/sse
EDP_MCP_STANDBY_URL=http://mcp-standby.example.com:8080/sse
EDP_MCP_ACCESS_TOKEN=your-mcp-token

# ============================================
# SPRING_PROFILES_ACTIVE=prod
```
