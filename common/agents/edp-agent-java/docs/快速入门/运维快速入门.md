# 运维快速入门

本文档将指导你快速完成 EDPAgent 的 Docker 部署、验证，并发送第一个任务。

## 前置条件

| 组件 | 要求 | 说明 |
|------|------|------|
| Docker | 20.10+ | 容器运行时 |
| Redis | 6.0+ | 任务状态存储，支持单机/哨兵/集群 |
| 大模型 API Key | - | 默认 DeepSeek，需有效 API Key |
| 可选组件 | Versatile、MCP SSE | 若需工作流调用和脚本沙箱 |

## 步骤一：准备 Redis

如果已有 Redis 服务可跳过此步。快速启动测试用 Redis：

```bash
docker run -d --name edp-redis -p 6379:6379 redis:7-alpine
```

> ⚠️ **生产环境提示**：测试用 Redis 无密码、无持久化，请勿用于生产。生产请使用独立部署的 Redis 集群或哨兵模式。

## 步骤二：获取 EDPAgent 镜像

### 方式一：从镜像仓库拉取（推荐）

```bash
docker pull <registry>/edp-agent-java:<version>
```

### 方式二：从源码构建

```bash
# 在代码仓根目录执行 Maven 打包
cd D:\AgentTool\EDPAgent_java\branch_edpa_java\agent-store\edp-agent-java
./mvnw clean install -DskipTests
./mvnw -pl engine -am package -DskipTests

# 构建 Docker 镜像
docker build -t edp-agent-java:latest -f deploy/Dockerfile .
```

## 步骤三：启动 EDPAgent 容器

```bash
docker run -d \
  --name edp-agent \
  -p 8190:8190 \
  -e EDP_AGENT_MODEL_API_KEY=your-deepseek-api-key \
  -e EDPA_REDIS_HOST=host.docker.internal \
  -e EDPA_REDIS_PORT=6379 \
  -e EDP_AGENT_SCENARIO_HOME=/app/scenarios/wealth-demo \
  edp-agent-java:latest
```

### 必填环境变量说明

| 环境变量 | 说明 |
|----------|------|
| `EDP_AGENT_MODEL_API_KEY` | **必须设置**，大模型 API Key，否则 Agent 无法推理 |
| `EDPA_REDIS_HOST` | Redis 主机地址，Docker for Windows/Mac 使用 `host.docker.internal` |
| `EDPA_REDIS_PORT` | Redis 端口，默认 6379 |
| `EDP_AGENT_SCENARIO_HOME` | 场景目录，Docker 中默认 `/app/scenarios/wealth-demo` |

> 📝 **提示**：若需要切换大模型（如使用私有部署的 OpenAI 兼容模型），需同时设置 `EDP_AGENT_MODEL_BASE_URL` 和 `EDP_AGENT_MODEL_NAME`。

## 步骤四：验证安装

### 1. 检查容器状态

```bash
docker ps | grep edp-agent
```

预期看到 `edp-agent` 容器状态为 `Up`。

### 2. 检查健康检查端点

```bash
curl http://localhost:8190/.well-known/agent-card.json
```

预期返回 Agent Card JSON，包含 agent 信息：

```json
{
  "agent": {
    "id": "edp-agent",
    "name": "通用动态规划智能体",
    "description": "..."
  }
}
```

Docker 内置 HEALTHCHECK 会每 10 秒访问此端点，连续 5 次失败则标记为 unhealthy。

### 3. 查看启动日志

```bash
docker logs edp-agent --tail 50
```

确认无 `ERROR` 级别日志。启动成功后会看到类似：

```
Started EdpaApplication in X.XXX seconds
```

## 步骤五：发送第一个任务

通过 A2A 协议发送测试消息（使用 SSE 流式响应）：

```bash
curl -N -X POST http://localhost:8190/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "test-001",
    "method": "sendStreamingMessage",
    "params": {
      "message": {
        "messageId": "msg-001",
        "role": "user",
        "parts": [{"kind": "text", "text": "你好，请介绍一下你自己"}]
      }
    }
  }'
```

如果一切正常，你将看到 SSE 事件流返回，格式如下：

```
event: message
data: {"jsonrpc":"2.0",...}

event: message
data: {"jsonrpc":"2.0",...}
```

最终会包含 Agent 的自我介绍文本："你好，我是通用动态规划智能体..."

## 常见问题

### 容器启动后立即退出

查看日志定位问题：

```bash
docker logs edp-agent
```

常见原因：
- Redis 连接失败：检查 `EDPA_REDIS_HOST` 和 `EDPA_REDIS_PORT` 是否正确
- 端口被占用：修改 `-p` 参数映射到其他端口
- API Key 无效：检查 `EDP_AGENT_MODEL_API_KEY`

### 调用 /a2a 返回错误

- 检查 Agent Card 是否能正常访问
- 确认 JSON 格式正确，特别是 `message.parts` 结构
- 查看容器日志获取详细错误信息

### Agent 回复异常

- 确认大模型 API Key 有效且有额度
- 检查网络是否能访问大模型 API 端点
- 查看日志中是否有模型调用超时或错误
