# skills — 金融精品 Skill

本目录用于存放 **金融行业 Agent 解决方案** 的精品 Skill 资产。

## 目录定位

`skills` 聚焦「能做什么」——将金融领域的专业知识与操作能力封装为 Agent 可直接加载、调用的 Skill，用于指导模型完成特定任务。

精品 Skill 应具备：

- **场景明确**：面向具体金融业务（如财报解读、合规问答、客户尽调等）
- **质量可控**：经过验证的提示词、流程与边界说明
- **可复用**：结构清晰，便于在不同 Agent 运行时中集成

## 与其他目录的关系

| 目录 | 职责 |
|------|------|
| `methodology/` | 方法论与流程工具——定义「如何分析与交付」 |
| `skills/` | 精品 Skill——封装面向 Agent 的可执行能力 |
| `mcp/` | 精品 MCP——对接外部系统与数据源 |

Skill 可在执行过程中引用 `methodology/` 中的框架与规范，并通过 `mcp/` 获取实时数据或调用外部服务。

## 目录约定

每个 Skill 建议以独立子目录组织，典型结构如下：

```
<skill-name>/
├── SKILL.md           # Skill 主文件（指令、触发条件、工作流）
├── scripts/           # 可选：配套脚本
├── references/        # 可选：参考文档
└── assets/            # 可选：静态资源
```

## 贡献说明

新增或更新 Skill 时，请确保：

1. `SKILL.md` 中说明适用场景、输入期望与输出格式
2. 明确能力边界，避免超出授权或合规范围
3. 遵循仓库 [贡献指南](../../CONTRIBUTING.md) 提交 PR
