# 提示词模板说明

## 概述

本 Skill 使用 YAML 格式的提示词模板，存放在 `assets/prompts/` 目录下。

## 模板文件

### generate_prompt.yaml

用于生成初始提示词的核心模板。

**占位符**：

- `{{ field_names_hint }}`：字段命名规范提示，当传入要素编码映射时会被替换

**模板内容**：

模板会指导 LLM 生成包含以下内容的提示词：

1. **角色设定**：针对该凭证类型的专家角色
2. **任务说明**：提取关键信息的任务描述
3. **提取指南**：每个字段的定位和提取方法
4. **JSON 输出格式**：期望的结构化输出示例

## 自定义模板

如需自定义提示词模板，可以修改 `assets/prompts/generate_prompt.yaml` 文件。

**注意事项**：

1. 保持 `{{ field_names_hint }}` 占位符，以便支持要素编码映射功能
2. 模板内容应使用中文，与 LLM 的理解能力匹配
3. 修改后无需重启服务，每次调用都会重新加载模板

## 要素编码映射

当传入 `field_code_mapping` 参数时，会生成特殊的字段命名提示，覆盖模板中默认的 JSON 命名规范。

**示例输入**：

```python
field_code_mapping = [
    {"code": "E01001", "name": "凭证编号"},
    {"code": "E01002", "name": "开票日期"},
]
```

**生成的提示会要求 LLM**：

1. 使用扁平列表格式，每个字段以"要素编码(中文名)"作为标识
2. JSON 输出使用要素编码作为 key，不使用嵌套结构

**示例输出格式**：

```json
{
  "E01001": "2024-001",
  "E01002": "2024-01-15"
}
```
