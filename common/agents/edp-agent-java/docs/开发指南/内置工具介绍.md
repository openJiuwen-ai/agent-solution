# 内置工具介绍

EDPAgent 内置了多个工具供 Agent 调用。工具分为两类：
- **框架原生工具**：由 DeepAgent 框架提供，包括 bash、skill_tool 以及 todo 系列工具
- **EDPAgent 业务工具**：由 EDPAgent 实现，包括 call_versatile、call_mcp、ask_user、cancel_task

本文档重点介绍 EDPAgent 的 4 个核心业务工具。Todo 工具和任务规划详见 [任务规划特性](../getting-started/features/任务规划特性.md)。

## 工具一览

| 工具名 | 类型 | 功能 | 触发中断 |
|--------|------|------|----------|
| `call_versatile` | 业务工具 | 委托 Versatile 执行工作流 | 是（等待用户交互时） |
| `call_mcp` | 业务工具 | 调用 MCP 沙箱执行脚本 | 是（等待脚本执行） |
| `ask_user` | 业务工具 | 向用户追问缺失信息 | 是（等待用户回复） |
| `cancel_task` | 业务工具 | 取消当前任务 | 否 |
| `todo_create` | 框架工具 | 创建 Todo 任务列表 | 否 |
| `todo_modify` | 框架工具 | 修改任务状态 | 否 |
| `todo_list` | 框架工具 | 列出所有任务 | 否 |
| `todo_get` | 框架工具 | 获取单个任务详情 | 否 |
| `skill_tool` | 框架工具 | 调用 Skill（内部使用） | 否 |
| `bash` | 框架工具 | 执行 Shell 命令 | 否 |

工具白名单在 actrule.yaml 的 `allowed_tools` 中配置。

---

## call_versatile

### 功能

委托外部 Versatile 服务执行业务工作流。VersatileInterruptRail 负责实际的 HTTP 调用、中断处理和结果归一化。

### 工具定义

- **工具名**：`call_versatile`
- **描述**：声明 Versatile Agent 委托意图，级联恢复和结果归一化由 VersatileInterruptRail 负责
- **返回值**：立即返回 `{"tool": "call_versatile", "status": "delegate_intent", "input": {...}}`，实际执行由 Rail 异步处理

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query_description` | string | 是 | 委托查询描述，说明本次调用要做什么 |
| `query_intent` | string | 是 | 委托查询意图，用于 Versatile 工作流路由 |
| `query_response_analysis_scripts` | array | 否 | 响应归一化脚本列表，对返回结果进行标准化处理 |
| `response_template_keys` | array | 否 | 响应话术模板 key 列表，从 SKILL.yaml/scriptconfig.yaml 中取话术 |
| `notice_context` | object | 否 | 非中断话术上下文，工作流执行过程中推送给用户的状态消息 |
| `input_key` | string | 否 | 从 ToolDataChannel 读取前序数据的 key |

### 调用示例

```json
{
  "name": "call_versatile",
  "arguments": {
    "query_description": "根据用户选择执行理财产品购买",
    "query_intent": "product_purchase",
    "input_key": "selected_product",
    "response_template_keys": ["purchase_success", "purchase_need_confirm"]
  }
}
```

### 使用约束

- 必须先创建 Todo 任务（todo_create）才能调用
- 同一时间只能有一个 Versatile 调用在执行
- 调用次数受 `tool_limits.call_versatile` 限制（默认50次/会话）
- 必须在 actrule.yaml 的 allowed_tools 中启用

### 中断恢复

当 Versatile 返回需要用户交互时，VersatileInterruptRail 会自动触发 ask_user 中断。用户回复后，重新调用 call_versatile（框架自动续传，无需 Agent 手动处理）。

## call_mcp

### 功能

调用 MCP SSE 服务在沙箱中执行 Python 脚本，用于数据查询、计算、格式转换等操作。

### 工具定义

- **工具名**：`call_mcp`
- **描述**：声明 MCP 沙箱脚本调用意图，执行和数据通道写入由 McpInterruptRail 负责
- **返回值**：立即返回 `{"tool": "call_mcp", "status": "mcp_intent", "input": {...}}`，实际执行由 Rail 异步处理

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `script_command` | string | 是 | 待执行的 MCP 脚本命令或脚本标识 |
| `script_params` | object | 否 | 脚本入参，key-value 格式 |

### 调用示例

```json
{
  "name": "call_mcp",
  "arguments": {
    "script_command": "recommend_products",
    "script_params": {
      "risk_level": "R2",
      "amount": 50000
    }
  }
}
```

### 结果处理

脚本执行完成后：
1. 结果自动写入 ToolDataChannel（键名：`mcp_result_{script_command}`）
2. 结果通过 SSE 事件推送给前端
3. 结果作为工具返回值给 LLM，LLM 可以观察到

### 使用约束

- 需要配置 `EDP_MCP_MASTER_URL`（可选配置备 URL）
- 调用次数受 `tool_limits.call_mcp` 限制（默认50次/会话）
- 必须在 actrule.yaml 的 allowed_tools 中启用

## ask_user

### 功能

向用户追问缺失信息或确认敏感操作。调用后触发中断，Agent 暂停执行等待用户回复。

### 工具定义

- **工具名**：`ask_user`
- **描述**：向用户追问缺失信息，并支持话术模板参数
- **返回值**：抛出 ToolInterruptException，不直接返回；用户回复后返回 `{"answer": "用户输入", ...}`

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `question` | string | 是 | 需要向用户追问的问题文本 |
| `response_template_keys` | array | 否 | 响应话术模板 key 列表 |
| `response_template_status` | string | 否 | 响应话术状态标识 |
| `response_template_vars` | object | 否 | 响应话术变量，用于模板占位符替换 |
| `missing_fields` | array | 否 | 缺失字段列表，便于前端渲染表单 |

### 调用示例

```json
{
  "name": "ask_user",
  "arguments": {
    "question": "好的，请问您要购买多少金额呢？",
    "missing_fields": ["amount"],
    "response_template_keys": ["amount_prompt"]
  }
}
```

### 中断恢复

用户回复后，工具返回格式：

```json
{
  "answer": "用户回复的文本内容",
  "interrupt_id": "ask_user_interrupt"
}
```

Agent 应从返回值中读取 `answer` 字段获取用户输入。

### 必须调用的场景

1. **关键参数缺失**：如购买金额未提供、收款账号未填
2. **敏感操作确认**：如购买确认、转账确认、取消确认
3. **用户输入有歧义**：如用户说"买那个"但未指明产品
4. **取消意图确认**：用户说"算了"、"不买了"时，先确认再取消

### 禁止行为

- 禁止在 final_answer 中提问，必须通过 ask_user 追问
- 禁止猜测用户未提供的参数
- 禁止反复追问相同问题
- 调用 cancel_task 前必须先 ask_user 确认

### 使用约束

- 调用次数受 `tool_limits.ask_user` 限制（默认50次/会话）
- 必须在 actrule.yaml 的 allowed_tools 中启用

## cancel_task

### 功能

取消当前任务，触发取消流程。将所有未完成的 Todo 标记为 CANCELLED，清理状态并返回取消话术。

### 工具定义

- **工具名**：`cancel_task`
- **描述**：取消当前任务并触发取消清理话术。使用前必须先通过 ask_user 向用户确认取消意图，获得用户明确确认后再调用此工具
- **返回值**：`{"tool": "cancel_task", "status": "cancelled", "input": {"reason": "..."}}`

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `reason` | string | 否 | 取消原因，记录在日志中 |

### 调用示例

```json
{
  "name": "cancel_task",
  "arguments": {
    "reason": "用户主动取消购买流程"
  }
}
```

### 标准取消流程

```
用户说"算了"
    │
    ▼
ask_user("确认取消当前操作吗？")
    │
    ├─ 用户说"是的" → cancel_task → 取消话术
    │
    └─ 用户说"继续" → 继续执行原任务
```

### 使用约束

- **必须先通过 ask_user 确认**用户的取消意图，不能直接取消
- reason 字段建议填写有意义的取消原因
- 取消后会话保持活跃，用户可发起新请求
- enable_task_loop 为 true 时，新请求会重新规划任务

## Todo 工具（框架原生）

Todo 工具由 DeepAgent 框架提供，用于管理任务状态。

### todo_create

创建任务列表。

**参数**：
```json
{
  "tasks": [
    {
      "catalog_id": "product_recommend",  // 使用预定义模板
      // 或手动指定：
      // "content": "任务内容",
      // "description": "任务详细描述"
    }
  ]
}
```

### todo_modify

修改任务状态。

**参数**：
```json
{
  "action": "update",  // update 或 cancel
  "task_id": "xxx",
  "status": "COMPLETED",  // PENDING / IN_PROGRESS / COMPLETED / CANCELLED
  "result_summary": "任务结果摘要"
}
```

**重要约束**：
- 调用时必须提供 action 参数
- 完成最后一个任务时，必须在同一迭代中调用 todo_modify 并输出 final_answer
- 不要重复调用相同的状态更新

### todo_list

获取所有任务列表，无参数。

### todo_get

获取单个任务详情。

**参数**：
```json
{
  "task_id": "xxx"
}
```

## 工具调用规约（来自 planrule.yaml）

1. **无依赖工具可并行调用**：同一迭代中可执行多个无依赖的工具调用（如 skill_tool + todo_create），但有数据依赖的工具必须分步执行。

2. **工具失败处理**：工具失败时在 thought 中记录原因，再决定重试或跳过；不编造数据。

3. **禁止并发工具调用（可配置）**：某些场景（如理财）通过 additional_prompt 强制要求每次只执行一个 tool_call。

4. **todo_modify 批量更新**：完成任务时可批量更新，一次 todo_modify 同时标记当前任务 COMPLETED + 下一任务 IN_PROGRESS。

5. **任务循环**：所有任务 COMPLETED/CANCELLED 后，用户有新意图时重新 todo_create，不需要手动处理历史任务。

## 工具注册机制

工具注册由 [EdpaToolRegistry.java](file:///D:/AgentTool/EDPAgent_java/branch_edpa_java/agent-store/edp-agent-java/engine/src/main/java/com/huawei/ascend/edp/tools/EdpaToolRegistry.java) 和 [EdpaBusinessTools.java](file:///D:/AgentTool/EDPAgent_java/branch_edpa_java/agent-store/edp-agent-java/engine/src/main/java/com/huawei/ascend/edp/tools/EdpaBusinessTools.java) 负责：

1. 从 actrule.allowed_tools 读取允许的工具列表
2. 跳过框架自动注册的工具（bash、skill_tool、todo_*）
3. 按名称查找对应构建器并实例化业务工具
4. 场景级可通过叠加 allowed_tools 新增自定义工具（未来 SPI 扩展）
