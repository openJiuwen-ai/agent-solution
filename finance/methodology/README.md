# methodology — 金融方法论工具

本目录用于存放 **金融行业 Agent 解决方案** 所依赖的方法论与工具资产。

## 目录定位

`methodology` 聚焦「怎么做」——提供可复用的分析框架、流程规范、评估模板与辅助工具，帮助 Agent 在合规前提下完成金融场景下的推理、决策与交付。

典型内容包括（但不限于）：

- **分析方法论**：如信贷评估、风险识别、合规审查、投研分析等标准流程与检查清单
- **决策框架**：评分模型说明、阈值定义、例外处理规则等
- **模板与表单**：报告结构、字段定义、输出格式约定
- **辅助工具脚本**：与方法论配套的校验、计算、格式化等小工具

## 与其他目录的关系

| 目录 | 职责 |
|------|------|
| `methodology/` | 方法论与流程工具——定义「如何分析与交付」 |
| `skills/` | 精品 Skill——封装面向 Agent 的可执行能力 |
| `mcp/` | 精品 MCP——对接外部系统与数据源 |

方法论是上层指导；Skill 与 MCP 在运行时引用或调用此处定义的规则与工具。

## 目录约定

每个方法论资产建议以独立子目录组织，并至少包含：

```
<methodology-name>/
├── README.md          # 方法论说明、适用场景与使用方式
├── ...                # 模板、脚本、参考文档等
```

1. 说明适用场景、输入输出与限制条件
