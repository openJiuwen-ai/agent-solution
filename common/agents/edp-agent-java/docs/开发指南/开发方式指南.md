# 开发方式指南

本文档介绍 EDPAgent 的两种开发方式：从零代码配置驱动到 Java 深度定制，帮助你根据业务需求选择合适的开发方式。

## 1. 概述

EDPAgent 支持两种开发方式，覆盖从快速体验到深度定制的完整需求：

| 开发方式 | 代码量 | 自定义能力 | 适用场景 |
|---------|--------|-----------|---------|
| **Layer 1: 配置驱动** | 零 Java 代码 | YAML 配置 + Skill 脚本 | 快速体验、独立部署、业务规则定制 |
| **Layer 2: Maven 集成** | Java 代码定制 | Layer 1 + 自定义 Tool/Rail | 客户系统集成、二次开发 |

两种方式可以混用：Layer 2 依赖 Layer 1 的配置体系，在配置驱动基础上叠加 Java 代码定制。

## 2. Layer 1: 配置驱动（零代码）

### 2.1 适用场景

- 快速体验 EDPAgent 能力
- 业务场景通过 YAML 配置即可定义（无需 Java 代码）
- 独立部署（引擎 + 场景配置一起打包）

### 2.2 工作方式

使用预构建的 JAR 包，配合外部 YAML 配置文件启动：

```
edp-agent-config-driven-sample/
├── application.yml          # 基础设施配置（端口/模型/Redis/沙箱）
├── start.bat / start.sh      # 启动脚本（java -jar 引擎 JAR）
└── scenarios/
    └── smart-customer-service/
        ├── governance/
        │   ├── planrule.yaml      # 角色规划（角色/范围/提示词/技能路由）
        │   ├── actrule.yaml       # 执行约束（任务模板/工具限制）
        │   └── scriptconfig.yaml  # 话术配置
        └── skills/
            └── faq_skill/          # 技能（YAML 声明 + Markdown 文档）
                ├── SKILL.yaml
                └── SKILL.md
```

**关键点**：
- 无 pom.xml，无需 Maven 构建
- 引擎 JAR 由 `edp-agent-java/engine` 模块构建（fat jar）
- 所有业务逻辑通过 governance YAML 声明
- 技能通过 SKILL.yaml + SKILL.md 声明（纯文本，LLM 运行时加载）

### 2.3 快速开始

```bash
# 1. 构建引擎 JAR（仅需一次）
cd edp-agent-java/engine
mvn clean package -DskipTests

# 2. 复制样例目录
cp -r common/example/edp-agent-config-driven-sample my-agent

# 3. 修改配置
#    application.yml: 设置模型 API Key
#    scenarios/*/governance/*.yaml: 定义角色和业务规则

# 4. 启动
cd my-agent
./start.sh    # Linux/Mac
start.bat     # Windows
```

### 2.4 参考样例

完整的 Layer 1 样例见 `common/example/edp-agent-config-driven-sample`，包含：
- 智能客服场景（smart-customer-service）
- 完整的 governance 三件套配置
- FAQ 技能示例
- 启动脚本和测试用例

## 3. Layer 2: Maven 集成 + Java 定制

### 3.1 适用场景

- 客户自有 Spring Boot 应用中嵌入 EDPAgent
- 需要注册自定义工具（Tool）和拦截器（Rail）
- 需要与其他系统集成（数据库、消息队列等）
- 二次开发和深度定制

### 3.2 工作方式

将 `edp-agent-engine` 作为 Maven 依赖引入客户应用，通过 Java 代码注册自定义扩展：

```
edp-agent-integration-example/
├── pom.xml                              # Maven POM（依赖 edp-agent-engine）
├── src/main/java/com/customer/agent/
│   ├── CustomerApplication.java          # 客户 Spring Boot 入口
│   └── CustomerAgentConfig.java          # Layer 2 定制（自定义 Tool + Rail）
├── src/main/resources/
│   └── application.yml                   # 客户配置
└── scenarios/
    └── wealth-demo/                      # 场景配置（同 Layer 1）
```

**关键点**：
- 使用 plain JAR（非 fat jar），依赖通过 Maven 传递
- 客户应用通过 `@SpringBootApplication(scanBasePackages=...)` 激活引擎自动配置
- 自定义代码通过 `@PostConstruct` 在引擎初始化后注册
- Layer 1 的 governance 配置体系完全保留

### 3.3 关键 API

Layer 2 定制的核心模式：

```java
@Configuration
public class CustomerAgentConfig {

    private final AgentHandler agentHandler;

    // 1. 注入 AgentHandler（由 EDPA 引擎自动注册，实际类型为 EdpaExtHandler）
    public CustomerAgentConfig(AgentHandler agentHandler) {
        this.agentHandler = agentHandler;
    }

    @PostConstruct
    public void registerCustomExtensions() {
        // 2. 强转为 EdpaExtHandler，获取 DeepAgent
        EdpaExtHandler edpaHandler = (EdpaExtHandler) agentHandler;
        DeepAgent deepAgent = edpaHandler.getDeepAgent();

        // 3. 注册自定义工具（LLM 可通过 function calling 调用）
        LocalFunction myTool = buildCustomTool();
        deepAgent.registerHarnessTool(myTool);

        // 4. 注册自定义 Rail（在模型调用前后注入逻辑）
        deepAgent.getAgent().registerRail(new CustomerAuditRail());
    }
}
```

#### 自定义工具（Tool）

使用 `LocalFunction` + `ToolCard` 构建自定义工具：

```java
// 构建 ToolCard（工具元数据：ID、名称、描述、参数 schema）
ToolCard card = ToolCard.builder()
        .id("my_tool")
        .name("my_tool")
        .description("My custom tool description")
        .inputParams(paramsSchema)  // JSON Schema 格式
        .build();

// 创建 LocalFunction（ToolCard + 执行逻辑）
LocalFunction tool = new LocalFunction(card, inputs -> {
    String param = (String) inputs.get("param");
    // 工具执行逻辑
    return Map.of("result", "done");
});

// 注册到 DeepAgent
deepAgent.registerHarnessTool(tool);
```

#### 自定义拦截器（Rail）

继承 `DeepAgentRail`，覆写回调方法：

```java
public class CustomerAuditRail extends DeepAgentRail {

    @Override
    public int priority() { return 15; }  // 数字越小越先执行

    @Override
    public void beforeModelCall(AgentCallbackContext ctx) {
        // 模型调用前逻辑（如审计日志、参数校验）
    }

    @Override
    public void afterModelCall(AgentCallbackContext ctx) {
        // 模型调用后逻辑（如响应处理、耗时统计）
    }
}

// 注册到 DeepAgent
deepAgent.getAgent().registerRail(new CustomerAuditRail());
```

#### Rail 优先级参考

| Rail | 优先级 | 职责 |
|------|--------|------|
| LogRail | 10 | 内置日志 |
| ScriptsRail | 50 | 话术注入 |
| ExecutionLimitRail | 70 | 执行步数限制 |
| EdpaEventRail | 80 | 事件发射 |
| McpInterruptRail | 85 | MCP 中断处理 |
| VersatileInterruptRail | 85 | Versatile 中断处理 |
| AskUserTemplateRail | 85 | 用户确认话术 |
| EdpaTodoRail | 95 | Todo 状态管理 |
| CancelRail | 100 | 取消请求处理 |

自定义 Rail 建议使用 priority 15-20（LogRail 之后，ScriptsRail 之前）。

### 3.4 快速开始

```bash
# 1. 安装引擎到本地 Maven 仓库
cd edp-agent-java/engine
mvn clean install -DskipTests

# 2. 复制样例目录
cp -r common/example/edp-agent-integration-example my-app

# 3. 修改 pom.xml（确认 agent-core-java 版本 >= 0.1.13）

# 4. 编写自定义代码
#    CustomerApplication.java: Spring Boot 入口
#    CustomerAgentConfig.java: 注册自定义 Tool/Rail

# 5. 构建和运行
cd my-app
mvn clean package -DskipTests
java -jar target/customer-agent-app-1.0.0.jar
```

## 4. 两种方式对比

| 对比项 | Layer 1: 配置驱动 | Layer 2: Maven 集成 |
|--------|------------------|-------------------|
| 使用方式 | `java -jar` 直接运行 fat jar | Maven 依赖嵌入自有应用 |
| JAR 类型 | fat jar（含全部依赖） | plain jar（依赖传递） |
| 代码量 | 零 Java 代码 | 有自定义 Java 代码 |
| 自定义能力 | YAML 配置 + Skill 脚本 | Layer 1 + 自定义 Tool/Rail |
| 构建工具 | 无需 Maven（使用预构建 JAR） | Maven |
| 适用场景 | 快速体验、独立部署 | 客户系统集成、二次开发 |
| 部署方式 | 引擎 + 场景配置一起部署 | 引擎嵌入客户应用一起部署 |

## 5. 选型建议

### 选择 Layer 1 的场景

- 业务逻辑通过 YAML 配置即可定义（角色、工具限制、话术）
- 不需要自定义 Java 工具或拦截器
- 快速验证 Agent 行为
- 独立部署（不嵌入其他应用）

### 选择 Layer 2 的场景

- 需要注册自定义工具（如调用内部 API、数据库查询）
- 需要自定义拦截器（如审计日志、安全校验、性能监控）
- 需要嵌入客户自有 Spring Boot 应用
- 需要与其他系统集成（数据库、消息队列等）

### 混用方式

Layer 2 在 Layer 1 基础上叠加，两者可以混用：
- Layer 2 应用仍然使用 governance YAML 配置（planrule/actrule/scriptconfig）
- Layer 2 的 Java 代码仅注册 YAML 无法表达的扩展（Tool/Rail）
- Skill 仍通过 SKILL.yaml + SKILL.md 声明
