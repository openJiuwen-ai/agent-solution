## 沙箱模式测试用例

> **范围**：EDPAgent sandbox + jiuwenbox + EvoAgentAdapter 三容器拓扑下，验证轨迹采集、调用代理、Skill 热更新及错误契约。  
> **资产**：`tests/sandbox_mode/`。  
> **不在范围**：共享挂载模式全量回归、EvoAgent 完整优化 Pipeline、managed-doc。

### 0. 测试层级定义（业界口径）

| 层级 | 定义 | 本目录判定 |
| --- | --- | --- |
| **单元（Unit）** | 测单个模块/类；外部依赖隔离 | pytest + mock，不启真实三容器 |
| **冒烟（Smoke）** | 部署后快速确认服务可达 | 仅 `/health`，不验业务协作 |
| **集成（Integration）** | 测组件间接口与协作 | 真实 HTTP；验 API/FS/契约；**不**要求完整用户旅程 |
| **系统（System）** | 在真实栈上验跨服务能力切片 | 含真实 LLM 的单次代理/轨迹，尚未构成完整业务故事 |
| **E2E（End-to-End）** | 类生产环境下完整业务路径 | 准备 → 对话 A → 热更 → 对话 B → 轨迹；断言用户可见结果 |

脚本按执行入口聚合，**层级以用例为准**，不以脚本文件名单独定义层级。

### 0.1 环境与依赖

| 项目 | 要求 |
| --- | --- |
| Python | 脚本 3.10+（标准库）；单元用 Adapter `.venv`（实测 3.13.13） |
| jiuwenbox | 默认 `http://127.0.0.1:8321` |
| EDPAgent | sandbox；健康检查 `http://127.0.0.1:18001/health` |
| Adapter | `skill_backend=jiuwenbox`；默认 `http://127.0.0.1:18900` |
| Skill | `product_recommend_skill`（沙箱 `/tmp/skills/...`） |
| 配置 | `ADAPTER_URL` / `EDP_URL` / `JIUWENBOX_URL` 或 CLI |

### 4. 集成（TC-04～07、10～14、17）

验组件接口协作与契约，**不以用户对话回答作为主断言**。

#### TC-04～07 Skill API ↔ 沙箱

| 编号 | 步骤要点 | 预期 |
| --- | --- | --- |
| TC-04 | `skill_list` | 含 `product_recommend_skill` |
| TC-05 | 先 `restore` 对齐基线，再 `skill_content` | 正文长度 > 50 |
| TC-06 | 注入唯一标记 → `update_skill` → `skill_content` | 读回含标记 |
| TC-07 | jiuwenbox `download` 同路径 | 文件含 TC-06 标记 |

#### TC-10～11 快照恢复

`restore_skill` 后与基线一致；再次 restore 仍成功（幂等）。

#### TC-12～14 错误契约

| 编号 | 预期 |
| --- | --- |
| TC-12 | 404 / `SKILL_NOT_FOUND` |
| TC-13 | 404 |
| TC-14 | 400 |

#### TC-17 热更写沙箱（集成）

`update_skill` 将内容改为 MARKER_B；jiuwenbox download 含 B、不含 A。  
（可在 E2E 脚本中作为场景中间步骤执行，**层级仍为集成**。）

### 6. E2E 场景（TC-E2E）

**场景名称**：沙箱模式下 Skill 热更对业务对话生效（无标记提示）。

**用户提问（A/B 相同，禁止写入 BEFORE/AFTER 字面量）：**

> 这是联调探针，不要做理财推荐。请立刻用 read_file 工具读取文件 `/tmp/skills/product_recommend_skill/SKILL.md`，在最终回答里原样输出文件中以 `HOTUPDATE_MARKER=` 开头的那一行（整行复制，不要改写）。不要调用 call_mcp/call_versatile/ask_user。

| 步骤 | 日志编号 | 角色 | 动作 | 预期 |
| --- | --- | --- | --- | --- |
| S1 | TC-15 | 准备 | 沙箱写入 MARKER_A | 沙箱文件含 A |
| S2 | TC-16 | E2E | 对话 A | 回答含 A、不含 B |
| S3 | TC-17 | 集成步骤 | `update_skill` → MARKER_B | 沙箱含 B、不含 A |
| S4 | TC-18 | E2E | 对话 B（同提问） | 回答含 B、不含 A |
| S5 | TC-19 | E2E 观测 | 查 A/B traces | 两会话 `total > 0` |

**E2E 门禁**：S2、S4、S5 必须通过；S1/S3 为场景必要步骤（S3 同时计入集成覆盖）。
