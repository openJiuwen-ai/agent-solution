# 技术支持渠道

## 遇到问题时

在寻求技术支持前，请先按以下顺序自查：

1. **查阅文档**
   - [常见问题 FAQ](常见问题.md) - 先看是否有你的问题
   - [故障排查手册](troubleshooting/服务启动失败.md) - 按症状排查
   - [运维文档](../operations/Docker部署指南.md) - 检查部署配置是否正确

2. **查看日志**
   - 收集启动日志和错误日志（容器名由 `docker run --name` 指定，或通过 `docker ps` 查看；application.yml 中 `openjiuwen.service.a2a.agent-name` 配置的是 A2A 协议中的 Agent 名称，默认为 "EDPAgent"）：
     ```bash
     # 查看正在运行的容器名
     docker ps --format "{{.Names}}"
     # 导出日志（将 <container-name> 替换为实际容器名）
     docker logs <container-name> --tail 200 > edp-agent-log.txt
     ```
   - 注意 ERROR 和 WARN 级别的日志
   - 开启 DEBUG 日志复现问题获取更详细信息

3. **基础检查**
   - 确认所有必填环境变量已正确配置（特别是 `EDP_AGENT_MODEL_API_KEY`）
   - 确认 Redis 服务正常运行且可连通
   - 确认大模型 API 可访问且 Key 有效
   - 确认 Docker/Java 版本满足要求

## 自助排错流程图

```
服务/请求异常
    │
    ├─ 服务能启动吗？
    │   ├─ 否 → [服务启动失败](troubleshooting/服务启动失败.md)
    │   │       ├─ 端口占用？→ 更换端口
    │   │       ├─ Redis连不上？→ [Redis连接问题](troubleshooting/Redis连接问题.md)
    │   │       ├─ YAML配置错误？→ 检查配置语法
    │   │       └─ JDK版本错误？→ 安装 JDK 21
    │   │
    │   └─ 是 → 健康检查通过吗？
    │           ├─ 否 → 查看日志 ERROR
    │           └─ 是 → 发送请求有响应吗？
    │                   ├─ 无响应/超时 → 检查大模型API [模型API问题](troubleshooting/模型API错误.md)
    │                   ├─ 返回业务错误 → 检查Versatile [Versatile问题](troubleshooting/Versatile调用异常.md)
    │                   ├─ 脚本错误 → [MCP问题](troubleshooting/MCP调用失败.md)
    │                   └─ 回复不符合预期 → 检查Governance配置/Skill触发
    │
    └─ 以上都不能解决 → 收集信息提交Issue
```

## 关键诊断信息收集

提交问题前，请收集以下信息：

### 1. 服务版本

```bash
# Docker 镜像版本（替换 <container-name> 为实际容器名）
docker inspect <container-name> --format '{{.Config.Image}}'

# Jar 版本
java -jar engine/target/edp-agent-engine-*.jar --version
```

### 2. 健康检查状态

```bash
curl -s http://localhost:8190/.well-known/agent-card.json
docker inspect <container-name> --format '{{json .State.Health}}'
```

### 3. 关键日志

```bash
# 最近的错误日志（替换 <container-name>）
docker logs <container-name> --tail 200 2>&1 | grep -E "ERROR|Exception|Failed"

# 启动日志（替换 <container-name>）
docker logs <container-name> 2>&1 | head -100
```

### 4. Redis 连接状态

```bash
redis-cli -h <host> -p <port> -a <password> PING
redis-cli INFO server | grep redis_version
```

### 5. 大模型连通性

```bash
curl -s -o /dev/null -w "%{http_code}" https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer $EDP_AGENT_MODEL_API_KEY"
```

### 6. 环境变量（脱敏）

```bash
# 列出关键环境变量，隐藏密钥值
env | grep -E "EDP_AGENT|EDPA_REDIS|EDP_MCP|EDP_AGENT_VERSATILE" | sed 's/=.*/=***/'
```
