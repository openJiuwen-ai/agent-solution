# mcp — 金融精品 MCP

本目录用于存放 **金融行业 Agent 解决方案** 的精品 MCP（Model Context Protocol）服务资产。

## 目录定位

`mcp` 聚焦「连什么、调什么」——提供与金融系统、数据源及第三方服务对接的标准化 MCP 实现，使 Agent 能够安全、可控地访问外部能力。

典型能力包括（但不限于）：

- **数据查询**：行情、财报、征信、监管公告等
- **业务系统对接**：核心银行、风控、CRM、工单等（在授权范围内）
- **工具集成**：计算引擎、文档解析、身份核验等

## 与其他目录的关系

| 目录 | 职责 |
|------|------|
| `methodology/` | 方法论与流程工具——定义「如何分析与交付」 |
| `skills/` | 精品 Skill——封装面向 Agent 的可执行能力 |
| `mcp/` | 精品 MCP——对接外部系统与数据源 |

MCP 为 Skill 与 Agent 运行时提供「手脚」；调用逻辑与合规约束应配合 `methodology/` 与 `skills/` 中的规范使用。

## 目录约定

每个 MCP 服务建议以独立子目录组织，典型结构如下：

```
<mcp-name>/
├── README.md          # 服务说明、配置方式与工具清单
├── ...                # 服务端实现、配置示例等
```

## 贡献说明

新增或更新 MCP 服务时，请确保：

1. 说明所需鉴权方式、环境变量与部署依赖
2. 明确数据范围、调用频率与错误处理约定
3. 不提交密钥、证书等敏感信息；敏感配置使用环境变量或密钥管理
4. 遵循仓库 [贡献指南](../../CONTRIBUTING.md) 提交 PR
