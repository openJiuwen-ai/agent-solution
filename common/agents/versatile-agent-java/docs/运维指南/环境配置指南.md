### VERSATILE_URL 占位符说明

`{conversation_id}` 是**必需**占位符：
- 运行时系统自动生成唯一 conversation_id
- 自动替换 URL 中所有 `{conversation_id}` 出现位置
- 手动测试连通性时需替换为测试值（如 `test-conv-123`）

URL 示例：
```
# 正确：包含 {conversation_id} 占位符
http://versatile:30001/v1/0/agent-manager/workflows/banking/conversations/{conversation_id}?type=controller&workspace_id=prod

# 错误：缺少占位符
http://versatile:30001/v1/0/agent-manager/workflows/banking/conversations/
```

## 6. 多意图 Endpoints 配置（application.yml 静态配置）

多意图路由通过 `endpoints` 列表配置，根据请求 metadata 中的 `intent` 字段路由到不同 URL 模板。

| 配置项 | 类型 | 配置路径 | 说明 |
|--------|------|----------|------|
| endpoints | List&lt;Endpoint&gt; | `openjiuwen.service.versatile.endpoints` | 按 intent 路由的端点配置列表 |

Endpoint 结构：
| 字段 | 类型 | 说明 |
|------|------|------|
| intent | String | 意图标识，匹配 A2A 请求 metadata 中的 `intent` 字段 |
| urlTemplate | String | 该 intent 使用的 Versatile URL 模板（同样需要包含 `{conversation_id}`） |

配置示例：
```yaml
openjiuwen:
  service:
    versatile:
      url-template: http://default-url/.../{conversation_id}
      endpoints:
        - intent: balance_query
          url-template: http://versatile/balance/.../{conversation_id}
        - intent: transfer
          url-template: http://versatile/transfer/.../{conversation_id}
```

**路由逻辑**：
1. 如果请求 metadata 中包含 `intent` 字段
2. 在 `endpoints` 列表中查找匹配 `intent` 的条目
3. 如果找到匹配，使用该条目的 `urlTemplate`
4. 如果没找到匹配，使用根级的 `urlTemplate` 作为默认

代码实现参见 [VersatileProperties.java:85-104](file:///D:/AgentTool/EDPAgent_java/agent-solution/common/agent-runtime-ext-java/agent-service-adapters/agent-service-adapters-versatile/src/main/java/com/openjiuwen/service/adapters/versatile/autoconfigure/VersatileProperties.java#L85-L104)。

> 默认 application.yml 未配置 endpoints，所有请求使用同一个 urlTemplate。

## 8. 配置优先级

```
高优先级 ────────────────────────────
  1. 操作系统环境变量 (-e xxx=yyy)
  2. docker run --env-file 传入的文件
  3. 挂载的外部配置 /app/config/application.yml
  4. jar 包内 application.yml
  5. 代码中的默认值
低优先级 ────────────────────────────
```

环境变量覆盖示例：
```bash
# 环境变量 VERSATILE_URL 会覆盖 application.yml 中的 url-template 配置
docker run -e VERSATILE_URL=http://custom-url/... adapter-versatile-agent-java
```

## 9. 配置示例

### 9.1 最小配置（仅必填项）

  --name versatile-agent \

### 9.2 生产环境配置模板 (docker-compose)

```yaml
version: '3.8'
services:
  versatile-agent:
    image: adapter-versatile-agent-java:latest
    container_name: versatile-agent
    restart: unless-stopped
    ports:
      - "8191:8191"
    environment:
      # ─── 时区 ───
      - TZ=Asia/Shanghai

      # ─── 端口 ───
      - VERSATILE_AGENT_PORT=8191

      # ─── Versatile 连接配置 ───
      - VERSATILE_URL=http://versatile-prod:30001/v1/0/agent-manager/workflows/banking_workflow/conversations/{conversation_id}?type=controller&workspace_id=production
      - VERSATILE_TIMEOUT=600s
      - VERSATILE_RESULT_NODE=GXZQAResponseNode

      # ─── JVM 调优（根据实际资源调整）───
      - JAVA_OPTS=-Xms512m -Xmx1024m -XX:+UseG1GC
    healthcheck:
      test: ["CMD", "curl", "-sf", "http://localhost:8191/.well-known/agent-card.json"]
      interval: 10s
      timeout: 5s
      retries: 8
      start_period: 90s
    logging:
      driver: json-file
      options:
        max-size: "100m"
        max-file: "10"
    networks:
      - agent-network
    # 如需自定义 Header/endpoints 配置，挂载外部配置：
    # volumes:
    #   - ./custom-application.yml:/app/config/application.yml:ro

networks:
  agent-network:
    driver: bridge
```

## 10. 快速验证配置是否生效

启动容器后，执行以下命令验证配置：

```bash
# 1. 查看容器运行中的环境变量
docker exec versatile-agent env | grep -E "VERSATILE|TZ|JAVA_OPTS"

# 2. 检查健康状态
curl -s http://localhost:8191/actuator/health

# 3. 检查 Agent Card（确认 Agent ID 正确）
curl -s http://localhost:8191/.well-known/agent-card.json | grep -E '"name"|"description"'

# 4. 查看启动日志确认无配置错误
docker logs --tail 50 versatile-agent
```
