# OpenClaw 安全加固工具包 v8 — 产品说明

> 版本：8.0.0 | 发布日期：2026-05-15

---

## 📖 这是什么？

**OpenClaw 安全加固工具包** 是一套专门为 **Rocky Linux 容器化部署的 OpenClaw** 设计的安全防护工具集。

简单来说：它帮你的 OpenClaw 系统「穿上铠甲」，自动完成从安全检查、漏洞修复、到效果验证的全流程，全程几乎不需要人工干预。

---

## 🎯 为什么需要这个工具包？

### 现实问题

OpenClaw 部署在生产环境后，面临以下安全风险：

| 风险 | 后果 |
|------|------|
| 默认 API Token 太短 | 容易被暴力破解，攻击者获得完全控制 |
| 配置文件权限宽松 | 服务器上其他用户可读取敏感信息 |
| Gateway 绑定公网 | 任何人都能直接访问你的 AI 助手 |
| 无安全审计机制 | 被入侵后无法发现，损失扩大 |
| 容器无安全限制 | 容器逃逸后可控制整台宿主机 |

### 传统加固方式的痛点

- ❌ **手动配置**：需要逐一检查几十个配置项，耗时且容易遗漏
- ❌ **缺乏标准**：没有统一的基线，每次加固效果不一致
- ❌ **无法回滚**：改错了不知道怎么恢复
- ❌ **没有验证**：加固后不确定是否真的生效
- ❌ **不可审计**：改了什么、效果如何，没有记录

### 本工具包的解决方案

- ✅ **全自动**：一条命令完成评估→加固→验证→报告
- ✅ **标准化**：L1-L5 五个安全等级，按需选择
- ✅ **可回滚**：加固前自动备份，一键恢复到加固前状态
- ✅ **可验证**：加固后逐项检查，确保每项都生效
- ✅ **可审计**：生成完整的 Markdown 报告，前后对比一目了然

## 🔧 核心功能

### 1. 智能预评估

在加固之前，工具包会全面扫描你的系统：

- 🔍 系统信息采集（OS、内核、容器环境）
- 🔍 OpenClaw 配置检查（Token 强度、绑定地址、权限）
- 🔍 敏感数据扫描（明文泄露的 API Key、密码）
- 🔍 容器安全状态（网络模式、只读根、资源限制）
- 📊 **输出安全评分**（0-100 分，分数越高越危险）

### 2. 分级自动加固

根据你的安全需求，选择 L1-L5 五个等级：

| 等级 | 适合谁 | 改什么 | 影响 |
|------|--------|--------|------|
| **L1 基础** | 开发测试环境 | Token、文件权限 | 很小 |
| **L2 标准** | 一般生产环境 | L1 + SSH + 日志 | 小 |
| **L3 增强** ⭐ | **推荐生产环境** | L2 + 防火墙 + 容器安全 | 中等 |
| **L4 严格** | 高安全要求 | L3 + 密码策略 + 审计增强 | 较大 |
| **L5 极限** | 合规要求最高 | L4 + 最严格限制 | 最大 |

**自动执行的加固项包括：**

- 🛡️ 生成强随机 API Token（如果当前太短）
- 🛡️ Gateway 绑定到 127.0.0.1（仅本机可访问）
- 🛡️ 修复所有文件和目录权限
- 🛡️ 创建 .env 文件分离敏感信息
- 🛡️ SSH 安全加固（禁用 root 密码登录、限制尝试次数）
- 🛡️ 开启防火墙，关闭无关服务
- 🛡️ 部署容器 seccomp 安全策略
- 🛡️ 注入 AI 安全规则到 SOUL.md / AGENTS.md
- 🛡️ 初始化文件完整性基线（防篡改监控）
- 🛡️ 配置每周自动安全审计

### 3. 加固后验证

加固完成后，自动验证每一项是否生效：

- ✅ Gateway 是否正常运行
- ✅ 端口是否正确监听
- ✅ 配置是否正确应用
- ✅ 文件权限是否到位
- ✅ 系统参数是否生效

### 4. 前后对比报告

自动生成 Markdown 格式的对比报告：

- 📊 加固前/后安全评分对比
- 📋 每项加固的通过/失败/跳过状态矩阵
- 📝 详细变更说明

### 5. 一键恢复（v8 新增）

如果加固后出现问题：

- 🔄 列出所有可用备份文件
- 🔄 支持全量恢复或选择性恢复（只恢复 SSH / OpenClaw / 系统参数）
- 🔄 **必须人工输入 YES 确认后才执行**（防止误操作）
- 🔄 恢复后自动验证是否恢复到加固前状态
- 🔄 生成恢复分析报告

## 📈 预期效果

### 安全评分改善

| 阶段 | 典型评分 | 状态 |
|------|----------|------|
| 加固前 | 30-60 分 | 🟡 中高风险 |
| 加固后 | 0-10 分 | 🟢 低风险 |

### 具体改善项

| 检查项 | 加固前 | 加固后 |
|--------|--------|--------|
| API Token 长度 | 可能 < 32 位 | 48 位强随机 |
| Gateway 绑定 | 0.0.0.0（公网） | 127.0.0.1（仅本机） |
| 配置文件权限 | 644（任何人可读） | 600（仅 owner） |
| SSH root 登录 | 允许密码登录 | 仅密钥登录 |
| 防火墙 | 可能未开启 | 仅开放必要端口 |
| 文件完整性 | 无监控 | SHA256 基线 + 每周审计 |
| AI 安全规则 | 默认配置 | 注入防注入/防泄露规则 |
| 容器安全 | 默认配置 | seccomp + 只读根 + 资源限制 |

## 🚀 使用场景

### ✅ 适合使用

| 场景 | 说明 | 推荐等级 |
|------|------|----------|
| **新服务器上线** | OpenClaw 刚部署好，需要从零加固 | L4（含 SSH） |
| **已运行但未加固** | OpenClaw 已在用，但没做过系统安全加固 | L3（--skip-ssh） |
| **合规审计前** | 需要安全报告应对审计或客户审查 | L4 + 导出全部报告 |
| **安全事件后** | 疑似被入侵，需要加固并恢复基线 | 先恢复，再 L4 加固 |
| **定期维护** | 按季度/半年度例行安全加固检查 | 跑预评估 + 增量加固 |
| **环境迁移** | 迁移到新服务器后重新建立安全基线 | L3（--skip-ssh） |

### ❌ 不适合

- Windows 环境（需要对应的 Windows 版本工具包）
- 非 Rocky Linux 发行版（部分脚本可能不兼容）
- 纯宿主机部署无容器（部分容器专属功能不适用）

## ⚠️ 加固后遗留风险及应对

### 重要说明

本工具包能解决 **OpenClaw 自身和运行环境** 的大部分安全问题，但 **不是万能药**。以下风险在加固后仍然存在，需要额外关注。

### 🔴 高风险遗留项

| 遗留风险 | 为什么没覆盖 | 潜在影响 | 应对建议 |
|----------|-------------|---------|----------|
| **AI 模型供应链风险** | 工具包只加固 OpenClaw 配置，不审查 LLM API 端的可信度 | 如果使用的模型 API 被劫持，可能返回恶意内容 | 仅使用官方/可信 API 端点，定期验证 API 端点域名 |
| **Prompt 注入绕过** | 工具包注入了防御规则，但高级注入可能绕过规则 | Agent 可能执行非预期操作 | 保持 AGENTS.md 安全规则更新，关注新型注入手法 |
| **第三方 Skill 漏洞** | 工具包只检查已安装 Skill，不主动扫描 | 恶意 Skill 可能窃取数据或执行危险命令 | 安装第三方 Skill 前审查源码，仅从 ClawHub 官方市场安装 |

### 🟡 中等风险遗留项

| 遗留风险 | 为什么没覆盖 | 潜在影响 | 应对建议 |
|----------|-------------|---------|----------|
| **操作系统层面漏洞** | 工具包加固 OpenClaw 相关配置，不修复系统 CVE | 已知 CVE 可被利用提权或远程执行 | 定期执行 `dnf update --security`，关注 Rocky 安全公告 |
| **容器镜像漏洞** | 工具包提供 seccomp 等运行时保护，不修复镜像本身 | 旧镜像中的 CVE 可能被利用 | 定期更新 OpenClaw 镜像版本，用 Trivy 扫描 |
| **网络层攻击** | 工具包配置防火墙，但不提供 DDoS 防护 | 大流量攻击可能导致服务不可用 | 如果公网暴露，使用 CDN/WAF 前置防护 |
| **会话劫持** | 工具包限制 SSH，不管理 OpenClaw 会话生命周期 | 过期会话可能被重放利用 | 定期清理旧会话文件，设置会话过期时间 |

### 🟢 低风险遗留项

| 遗留风险 | 为什么没覆盖 | 潜在影响 | 应对建议 |
|----------|-------------|---------|----------|
| **内部人员滥用** | 工具包是技术加固，不解决管理和流程问题 | 有权限的人可能恶意操作 | 建立操作审批流程，限制 root 账号使用 |
| **物理安全风险** | 工具包是软件层面加固 | 物理接触服务器可直接获取数据 | 确保服务器机房安全，启用磁盘加密（LUKS） |
| **日志外发缺失** | 工具包配置本地日志审计，不提供日志远程备份 | 服务器被攻陷后日志可能被篡改或删除 | 配置 rsyslog 将日志发送到独立的日志服务器 |

## 📋 使用建议

### 首次使用

1. **先在测试环境演练** — 跑一遍完整流程，确认所有脚本正常工作
2. **先跑 `--dry-run`** — 看一遍脚本要改什么，确认无误再执行
3. **保持 SSH 连接不关** — 加固期间不要断开当前 SSH 会话
4. **加 `--skip-ssh`** — 已能正常登录的话，不要改 SSH 配置

### 生产环境

1. **选择 L3 或 L4 等级** — L3 适合大多数场景，L4 适合高安全要求
2. **保留所有报告** — 用于审计和故障排查
3. **定期检查审计日志** — 每周自动审计的日志在 `/var/log/openclaw-audit.log`

## ⛔ 操作风险提示

以下是在 **执行加固过程中** 可能遇到的操作风险（与上面的"遗留安全风险"不同）：

| 风险 | 说明 | 应对 |
|------|------|------|
| SSH 无法登录 | 加固可能修改 SSH 配置 | 加 `--skip-ssh` 避免；或保持 SSH 会话不断 |
| OpenClaw 重启 | 配置变更后需要重启 Gateway | 选择维护窗口执行 |
| 容器不兼容 | 某些安全配置可能与容器冲突 | 脚本会自动检测容器模式并跳过不适用的项 |
| 恢复失败 | 备份文件损坏或路径不匹配 | 加固前建议手动额外备份关键配置 |
