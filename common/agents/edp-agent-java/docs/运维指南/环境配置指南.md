# 环境变量配置详解

EDPAgent 所有配置均可通过环境变量覆盖，无需修改配置文件或 Jar 包内的文件。本文档详细说明每个环境变量的用途和配置方法。

## 配置加载机制

Spring Boot 使用 `${ENV_VAR:default}` 占位符语法，环境变量优先级高于 application.yml 中的默认值：

```
环境变量 > application.yml 默认值
```

Docker 部署时通过 `-e` 或 `--env-file` 传入环境变量；本地启动时可通过 shell 环境变量或 IDEA Run Configuration 设置。

## Redis 连接配置

### 基础连接（Spring Data Redis）

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDPA_REDIS_HOST` | `localhost` | 是 | Redis 主机地址 |
| `EDPA_REDIS_PORT` | `6379` | 否 | Redis 端口 |
| `EDPA_REDIS_PASSWORD` | （空） | 否 | Redis 密码，无密码不设置 |
| `EDPA_REDIS_DB` | `0` | 否 | Redis 数据库编号 |

### Redis 模式与高级配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDPA_REDIS_MODE` | `single` | 否 | Redis 部署模式：`single`（单机）/ `sentinel`（哨兵）/ `cluster`（集群） |
| `EDPA_REDIS_CONNECT_TIMEOUT` | `5000` | 否 | 连接超时时间（毫秒） |
| `EDPA_REDIS_SOCKET_TIMEOUT` | `10000` | 否 | Socket 读写超时时间（毫秒） |
| `EDPA_REDIS_CHECKPOINTER_TTL` | `60` | 否 | Checkpoint（中断恢复点）数据 TTL（分钟） |
| `EDPA_REDIS_KEY_PREFIX` | `edpa` | 否 | Todo 数据 Key 前缀，多实例共用 Redis 时用于区分 |
| `EDPA_REDIS_TODO_TTL` | `3600` | 否 | Todo 任务数据 TTL（秒），默认1小时 |
| `EDPA_REDIS_REFRESH_ON_READ` | `true` | 否 | 读取 Todo 时是否自动续期 TTL |

### Redis 配置示例

**单机无密码（默认）**：
```bash
EDPA_REDIS_HOST=localhost
EDPA_REDIS_PORT=6379
```

**单机有密码**：
```bash
EDPA_REDIS_HOST=redis.example.com
EDPA_REDIS_PORT=6379
EDPA_REDIS_PASSWORD=your-redis-password
EDPA_REDIS_DB=0
```

**Docker 环境连宿主机 Redis**：
```bash
# Windows/Mac Docker Desktop
EDPA_REDIS_HOST=host.docker.internal
# Linux Docker（需要 --network host 或使用宿主机IP）
EDPA_REDIS_HOST=172.17.0.1
```

**生产环境建议**：
- 生产环境 Redis 设置密码
- 根据网络情况调整超时时间（跨机房部署建议增大到 10000ms）
- Checkpointer TTL 根据业务需要调整（如需支持用户隔天继续操作，可增大到 1440 分钟即 24 小时）
- Todo TTL 建议保持默认或适当增大

## Agent 核心配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDPA_AGENT_CONFIG_PATH` | `src/main/resources/edp-config.yaml` | 否 | 配置锚点文件路径。Docker 中默认为 `/app/config/edp-config.yaml`，一般不需要修改 |
| `EDP_AGENT_SCENARIO_HOME` | `../scenarios/wealth-demo` | 是 | **场景目录路径**——切换此值即可切换业务场景 |

### EDP_AGENT_SCENARIO_HOME 说明

这是最常用的配置之一，用于切换业务场景：

```bash
# 使用理财演示场景（默认）
EDP_AGENT_SCENARIO_HOME=/app/scenarios/wealth-demo

# 使用智贷通场景
EDP_AGENT_SCENARIO_HOME=/app/scenarios/hz-zhidaitong

# 使用自定义场景（需要挂载到容器内）
EDP_AGENT_SCENARIO_HOME=/app/scenarios/my-custom-scenario
```

Docker 镜像中默认打包了 wealth-demo 场景。如需使用其他场景，需将场景目录挂载到容器中。

## 大模型配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_AGENT_MODEL_PROVIDER` | `OpenAI` | 否 | 模型提供者，OpenAI 兼容接口统一填 `OpenAI` |
| `EDP_AGENT_MODEL_NAME` | `deepseek-v4-pro` | 否 | 模型名称 |
| `EDP_AGENT_MODEL_BASE_URL` | `https://api.deepseek.com/v1` | 否 | 模型 API Base URL |
| `EDP_AGENT_MODEL_API_KEY` | （见 application.yml 默认值） | **是** | **必填**：大模型 API Key |

> ⚠️ **重要**：`EDP_AGENT_MODEL_API_KEY` 必须设置为有效的 API Key，否则 Agent 无法进行推理，所有请求都会失败。

### 切换大模型示例

**使用 DeepSeek（默认）**：
```bash
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=deepseek-v4-pro
EDP_AGENT_MODEL_BASE_URL=https://api.deepseek.com/v1
EDP_AGENT_MODEL_API_KEY=sk-your-deepseek-key
```

**使用 OpenAI GPT-4o**：
```bash
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=gpt-4o
EDP_AGENT_MODEL_BASE_URL=https://api.openai.com/v1
EDP_AGENT_MODEL_API_KEY=sk-your-openai-key
```

**使用私有部署模型（vLLM/Ollama 等）**：
```bash
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=your-model-name
EDP_AGENT_MODEL_BASE_URL=http://your-llm-server:8000/v1
EDP_AGENT_MODEL_API_KEY=your-key-if-needed
```

## Versatile 工作流配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_AGENT_VERSATILE_URL` | `http://localhost:30001/v1/0/agent-manager/workflows/{workflow_id}/conversations/{conversation_id}` | 否 | Versatile REST API 地址（含路径变量模板） |
| `EDP_AGENT_VERSATILE_A2A_URL` | `http://localhost:8191/a2a` | 否 | Versatile A2A 适配器地址 |
| `EDP_AGENT_VERSATILE_TIMEOUT` | `30s` | 否 | Versatile 调用超时时间 |

### URL 模板变量

URL 中的 `{workflow_id}` 和 `{conversation_id}` 是路径变量，运行时自动填充：
- `{workflow_id}`：工作流 ID，默认 `mock_workflow`，场景可配置路由
- `{conversation_id}`：当前会话 ID，由框架自动生成

### 超时格式

超时时间支持 Spring Duration 格式：
- `30s` = 30秒
- `1m` = 1分钟
- `60s` = 60秒

生产环境建议设置为 60s 或更长，复杂工作流可能耗时较久。

## MCP SSE 配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_MCP_MASTER_URL` | （空） | 否 | MCP SSE 主服务 URL |
| `EDP_MCP_STANDBY_URL` | （空） | 否 | MCP SSE 备服务 URL（主 URL 不可用时自动切换） |
| `EDP_MCP_ACCESS_TOKEN` | （空） | 否 | MCP SSE 鉴权 Token |
| `EDP_MCP_APP_NAME` | （空） | 否 | MCP 应用名称标识 |

如果场景不使用 `call_mcp` 工具，这些都可以不配置。使用主备模式时建议同时配置 master 和 standby。

## 日志配置

| 环境变量 | 默认值 | 必填 | 说明 |
|----------|--------|------|------|
| `EDP_AGENT_LOG_LEVEL` | `INFO` | 否 | 日志级别：`DEBUG`/`INFO`/`WARN`/`ERROR` |

### 日志级别说明

| 级别 | 适用场景 | 日志量 |
|------|----------|--------|
| `DEBUG` | 开发调试、问题排查 | 非常详细，包含 Prompt 内容、工具调用参数 |
| `INFO` | 生产环境默认 | 关键操作记录，启动信息，警告错误 |
| `WARN` | 高并发生产环境 | 仅警告和错误 |
| `ERROR` | 仅错误排查 | 只记录错误 |

生产环境建议使用 `INFO` 级别，排查问题时临时切换到 `DEBUG`。

## 环境变量配置示例

### 最小配置（仅必填项）

```bash
# 大模型 API Key（必须修改）
EDP_AGENT_MODEL_API_KEY=sk-your-api-key-here

# Redis 地址（如果 Redis 在本机）
EDPA_REDIS_HOST=localhost

# 场景
EDP_AGENT_SCENARIO_HOME=/app/scenarios/wealth-demo
```

### 完整生产配置

```bash
# 服务端口
SERVER_PORT=8190

# Redis 配置
EDPA_REDIS_MODE=single
EDPA_REDIS_HOST=redis.internal.example.com
EDPA_REDIS_PORT=6379
EDPA_REDIS_PASSWORD=your-production-redis-password
EDPA_REDIS_DB=0
EDPA_REDIS_CONNECT_TIMEOUT=5000
EDPA_REDIS_SOCKET_TIMEOUT=10000
EDPA_REDIS_CHECKPOINTER_TTL=1440
EDPA_REDIS_KEY_PREFIX=edpa-prod
EDPA_REDIS_TODO_TTL=7200
EDPA_REDIS_REFRESH_ON_READ=true

# Agent 配置
EDP_AGENT_SCENARIO_HOME=/app/scenarios/wealth-demo

# 大模型配置
EDP_AGENT_MODEL_PROVIDER=OpenAI
EDP_AGENT_MODEL_NAME=deepseek-v4-pro
EDP_AGENT_MODEL_BASE_URL=https://api.deepseek.com/v1
EDP_AGENT_MODEL_API_KEY=sk-your-production-key

# Versatile 配置
EDP_AGENT_VERSATILE_URL=http://versatile.internal.example.com/v1/0/agent-manager/workflows/{workflow_id}/conversations/{conversation_id}
EDP_AGENT_VERSATILE_TIMEOUT=60s

# MCP 配置
EDP_MCP_MASTER_URL=http://mcp-master.internal.example.com:8080/sse
EDP_MCP_STANDBY_URL=http://mcp-standby.internal.example.com:8080/sse
EDP_MCP_ACCESS_TOKEN=your-mcp-token
EDP_MCP_APP_NAME=edp-agent-prod

# 日志
EDP_AGENT_LOG_LEVEL=INFO
```

### 本地开发配置

```bash
# Redis（Docker 启动的本地 Redis）
EDPA_REDIS_HOST=localhost
EDPA_REDIS_PORT=6379

# 场景（本地源码路径）
EDP_AGENT_SCENARIO_HOME=../scenarios/wealth-demo

# 大模型
EDP_AGENT_MODEL_API_KEY=sk-your-dev-key

# Mock Versatile
EDP_AGENT_VERSATILE_URL=http://localhost:30001/v1/0/agent-manager/workflows/{workflow_id}/conversations/{conversation_id}

# 日志：开发环境用 DEBUG
EDP_AGENT_LOG_LEVEL=DEBUG
```

## .env 文件模板

代码仓根目录提供了 [.env.example](file:///D:/AgentTool/EDPAgent_java/branch_edpa_java/agent-store/edp-agent-java/.env.example) 文件作为模板：

```bash
cp .env.example .env
# 编辑 .env 填入你的配置
```

.env 文件可配合 docker-compose 使用，或通过 shell 的 `source .env` / `export $(cat .env | xargs)` 加载。

## 配置验证

启动服务后，通过以下方式验证配置是否生效：

1. **查看启动日志**：日志中会打印加载的配置信息
2. **访问 Agent Card**：返回的 Agent 信息反映当前配置
3. **发送测试消息**：验证大模型调用正常
4. **检查 Redis 连接**：日志中无 Redis 连接错误

常见配置错误见 [故障排查 - 服务启动失败](../support/troubleshooting/服务启动失败.md)。
