# 开发环境准备

本文档介绍如何搭建 EDPAgent 的本地开发环境，用于配置 Governance 规则、开发 Skill、调试场景。

## 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 17+ | Spring Boot 4 要求 Java 17+ |
| Maven | 3.8+ | 项目构建工具 |
| Python | 3.10+ | Skill 脚本运行环境 |
| Redis | 5.0+ | 任务状态存储（单机即可） |
| Git | - | 版本管理 |
| IDE | IntelliJ IDEA 推荐 | Java/Python 开发 |

## 步骤一：安装基础依赖

### Windows

```powershell
# 使用 Chocolatey 安装（管理员权限）
choco install openjdk21 maven python redis

# 或手动下载安装：
# JDK 21: https://adoptium.net/
# Maven: https://maven.apache.org/download.cgi
# Python: https://www.python.org/downloads/
# Redis (Windows): 使用 Docker 或 Memurai
```

### macOS

```bash
# 使用 Homebrew
brew install openjdk@21 maven python redis

# 启动 Redis
brew services start redis
```

### Linux (Ubuntu/Debian)

```bash
# 安装 JDK 21
sudo apt install openjdk-21-jdk maven python3 python3-pip redis-server

# 启动 Redis
sudo systemctl start redis-server
sudo systemctl enable redis-server
```

## 步骤二：获取代码

```bash
git clone <repository-url>
cd edp-agent-java
```

## 步骤三：使用 Docker 启动 Redis（推荐）

如果不想在本地安装 Redis，可以使用 Docker 快速启动：

```bash
docker run -d --name edp-redis -p 6379:6379 redis:7-alpine
```

验证 Redis 连接：

```bash
# 使用 redis-cli 连接
docker exec -it edp-redis redis-cli ping
# 预期返回 PONG
```

## 步骤四：配置环境变量

在项目根目录创建 `.env` 文件（参考 `.env.example`）：

```bash
# 复制模板
cp .env.example .env

# 编辑 .env 文件，设置必填项
```

必填配置：

```bash
# 大模型 API Key（必须配置，否则无法推理）
EDP_AGENT_MODEL_API_KEY=your-deepseek-api-key

# Redis 配置（Docker 启动默认即可）
EDPA_REDIS_HOST=localhost
EDPA_REDIS_PORT=6379

# 使用理财演示场景
EDP_AGENT_SCENARIO_HOME=../scenarios/wealth-demo

# 日志级别（开发环境建议 DEBUG）
EDP_AGENT_LOG_LEVEL=DEBUG
```

> ⚠️ **注意**：`.env` 文件仅用于本地开发，不要提交到代码仓。生产环境通过容器环境变量传入。

## 步骤五：启动服务

### 方式一：Maven 命令行启动

```bash
cd engine
./mvnw spring-boot:run
```

Windows 下使用：

```powershell
cd engine
.\mvnw.cmd spring-boot:run
```

### 方式二：IDEA 中启动

1. 用 IDEA 打开项目根目录的 `pom.xml`，以 Maven 项目导入
2. 等待依赖下载完成
3. 找到 `engine/src/main/java/com/huawei/ascend/edp/EdpaApplication.java`
4. 右键 → Run 'EdpaApplication'

### 方式三：先打包再运行

```bash
# 在项目根目录执行
./mvnw clean install -DskipTests
./mvnw -pl engine -am package -DskipTests

# 运行打包后的 jar
java -jar engine/target/edp-agent-engine-*.jar
```

## 步骤六：验证服务启动

服务启动后（默认端口 8190）：

```bash
# 检查 Agent Card
curl http://localhost:8190/.well-known/agent-card.json

# 发送测试消息
curl -N -X POST http://localhost:8190/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "dev-test-001",
    "method": "sendStreamingMessage",
    "params": {
      "message": {
        "messageId": "msg-001",
        "role": "user",
        "parts": [{"kind": "text", "text": "你好"}]
      }
    }
  }'
```

看到 SSE 事件流返回即表示服务正常运行。

## Python 环境配置（MCP 脚本开发用）

如果需要开发和调试 Skill 中的 Python 脚本，配置 Python 环境：

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 安装 MCP 依赖
pip install -r deploy/requirements-mcp.txt

# 安装开发/调试工具
pip install requests pytest
```

## 使用 Mock 服务器（可选）

开发时可以使用代码仓自带的 Mock Versatile 服务器，避免依赖真实的 Versatile 服务：

```bash
cd mock
pip install -r requirements.txt  # 如有需要
python versatile_main.py
```

Mock 服务器默认启动在 30001 端口，提供模拟的工作流响应。同时需要配置环境变量指向 Mock：

```bash
EDP_AGENT_VERSATILE_URL=http://localhost:30001/v1/0/agent-manager/workflows/{workflow_id}/conversations/{conversation_id}
```

## IDEA 调试技巧

1. **热重载**：安装 Spring Boot DevTools 或使用 JRebel，修改 Java 代码后自动重启（YAML 配置修改仍需手动重启）。
2. **断点调试**：在关键 Rail 或工具代码处设置断点，Debug 模式启动。
3. **日志级别**：开发环境设置 `EDP_AGENT_LOG_LEVEL=DEBUG`，可以看到详细的工具调用和 Rail 执行日志。
4. **Redis 数据查看**：使用 RedisInsight 或 `redis-cli` 查看 Todo 数据和 Checkpoint：
   ```bash
   docker exec -it edp-redis redis-cli
   keys edpa:*  # 查看所有 EDPAgent 相关 key
   ```

## 常见开发问题

### Maven 依赖下载慢

配置国内 Maven 镜像，在 `~/.m2/settings.xml` 中添加阿里云镜像。

### Python 脚本执行失败

- 确认 Python 版本 3.10+
- 确认已安装 requirements-mcp.txt 中的依赖
- 查看 Agent 日志中的 Python 错误堆栈
- 本地开发时可直接运行脚本测试：`python scripts/run_xxx.py '{"key":"value"}'`

### 配置修改后不生效

- 当前版本配置修改后**必须重启服务**，不支持热加载
- 确认 `EDP_AGENT_SCENARIO_HOME` 路径正确，场景配置在正确的目录下
- 检查日志中是否有配置加载错误

### Redis 连接失败

- 确认 Redis 服务已启动：`docker ps | grep redis`
- 确认 `EDPA_REDIS_HOST` 和 `EDPA_REDIS_PORT` 配置正确
- Docker for Windows/Mac 连接宿主机 Redis 使用 `host.docker.internal`
