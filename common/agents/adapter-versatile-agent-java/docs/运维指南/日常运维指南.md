# Versatile Agent 日常运维指南

本文档介绍 adapter-versatile-agent-java 服务的日常运维操作。

---

## 1. 服务启停

假设容器名为 `versatile-agent`。

### 1.1 启动服务

```bash
# 启动已停止的容器
docker start versatile-agent

# 如果是 docker-compose
docker-compose up -d
```

### 1.2 停止服务

```bash
# 优雅停止
docker stop versatile-agent

# 如果是 docker-compose
docker-compose stop
```

### 1.3 重启服务

```bash
# 重启容器
docker restart versatile-agent

# 如果是 docker-compose
docker-compose restart
```

### 1.4 查看容器状态

```bash
# 查看运行状态
docker ps -a --filter name=versatile-agent

# 查看详细信息（含健康状态）
docker inspect versatile-agent
```

## 3. 健康检查

### 3.1 通过 Actuator 端点检查

```bash
# 健康检查端点
curl http://localhost:8191/actuator/health
```

正常返回：
```json
{"status":"UP"}
```

异常返回：
```json
{"status":"DOWN"}
```

### 3.2 通过 Docker 检查

```bash
# 查看健康状态
docker inspect --format='{{.State.Health.Status}}' versatile-agent

# 查看健康检查历史记录
docker inspect --format='{{range .State.Health.Log}}{{.ExitCode}} - {{.Output}}{{"\n"}}{{end}}' versatile-agent
```

状态说明：
- `healthy`：服务正常
- `unhealthy`：连续多次健康检查失败
- `starting`：服务正在启动中（90秒内）

### 3.3 通过 Agent Card 验证

```bash
curl http://localhost:8191/.well-known/agent-card.json
```

此端点为 Docker HEALTHCHECK 使用的检查端点（参见 [Dockerfile:24-25](file:///D:/AgentTool/EDPAgent_java/agent-solution/common/agent/adapter-versatile-agent-java/deploy/Dockerfile#L24-L25)），能正常返回说明：
- 服务端口正常监听
- Spring Boot 应用启动完成
- A2A Agent Card 端点可用

## 4. Agent Card 验证

Agent Card 是 A2A 协议的基础元数据端点，部署后必须验证。

```bash
curl -s http://localhost:8191/.well-known/agent-card.json | python -m json.tool
```

验证点：
- `name` 字段应为 `"versatile-agent"`
- `description` 包含服务描述

## 5. 配置变更

### 5.1 修改环境变量

环境变量变更需要**重建容器**（Docker 不支持修改运行中容器的环境变量）：

```bash
# 1. 停止并删除旧容器
docker stop versatile-agent
docker rm versatile-agent

# 2. 使用新环境变量启动新容器
docker run -d \
  --name versatile-agent \
  -p 8191:8191 \
  -e VERSATILE_URL="http://new-versatile-url/..." \
  -e VERSATILE_TIMEOUT=900s \
  adapter-versatile-agent-java:latest
```

### 5.2 挂载外部配置文件

如需修改 application.yml 中的 Header、endpoints 等配置，推荐挂载外部配置文件：

```bash
# 将自定义配置挂载到 /app/config/application.yml
docker run -d \
  --name versatile-agent \
  -p 8191:8191 \
  -v /path/to/custom-application.yml:/app/config/application.yml:ro \
  adapter-versatile-agent-java:latest
```

Spring Boot 会优先加载 `/app/config/application.yml` 中的配置。

### 5.3 验证配置生效

```bash
# 查看容器启动时使用的环境变量
docker exec versatile-agent env | grep VERSATILE

# 检查配置后通过健康检查确认服务重启成功
curl http://localhost:8191/actuator/health
```

## 6. 版本升级

本服务**无状态**，不依赖 Redis，版本升级可直接替换容器，无需数据迁移。

### 6.1 升级步骤

```bash
# 1. 构建新版本镜像
mvn clean package -DskipTests
docker build -t adapter-versatile-agent-java:v2 -f deploy/Dockerfile .

# 2. 停止旧容器
docker stop versatile-agent

# 3. （可选）备份旧容器日志
docker logs versatile-agent > upgrade-backup-$(date +%Y%m%d).log 2>&1

# 4. 删除旧容器
docker rm versatile-agent

# 5. 启动新容器（使用相同的环境变量和端口映射）
docker run -d \
  --name versatile-agent \
  -p 8191:8191 \
  -e VERSATILE_URL="http://your-versatile-url/..." \
  adapter-versatile-agent-java:v2

# 6. 验证新版本
curl http://localhost:8191/actuator/health
curl http://localhost:8191/.well-known/agent-card.json
```

### 6.2 快速回滚

如果新版本有问题，直接重启旧容器（如果镜像还在）：

```bash
# 停止新容器
docker stop versatile-agent
docker rm versatile-agent

# 启动旧版本容器
docker run -d \
  --name versatile-agent \
  -p 8191:8191 \
  -e VERSATILE_URL="http://your-versatile-url/..." \
  adapter-versatile-agent-java:v1
```

## 7. 常见运维操作

### 7.1 查看连接数/端口监听

```bash
# 进入容器
docker exec -it versatile-agent sh

# 检查端口监听
netstat -tlnp | grep 8191
# 或
ss -tlnp | grep 8191
```

或在宿主机检查：
```bash
# 检查端口映射
docker port versatile-agent
```

### 7.2 检查 Versatile 服务连通性

**重要**：将 URL 中的 `{conversation_id}` 替换为一个测试值（如 `test-conv-123`）：

```bash
# 在容器内测试 Versatile 连通性
docker exec -it versatile-agent sh
curl -v "http://your-versatile-host:port/path/conversations/test-conv-123?type=controller&workspace_id=10"

# 或直接在宿主机测试（如果网络可达）
curl -v "http://your-versatile-host:port/path/conversations/test-conv-123?type=controller&workspace_id=10"
```

连通性测试要点：
1. URL 中 `{conversation_id}` 必须替换为实际字符串，不能保留占位符
2. 检查是否能建立 TCP 连接
3. 检查 HTTP 响应状态码
4. 如果返回 404 检查路径是否正确，返回 401/403 检查是否需要认证

### 7.3 检查容器资源使用

```bash
# 查看容器资源占用
docker stats versatile-agent --no-stream

# 查看容器内进程
docker top versatile-agent
```

### 7.4 进入容器调试

```bash
# 进入容器 shell
docker exec -it versatile-agent sh

# 查看容器内文件
ls -la /app
ls -la /app/logs/run/

# 如果容器未启动成功，查看启动日志
docker logs versatile-agent
```

### 7.5 清理日志

如果使用 json-file 日志驱动且未配置轮转，可清理旧日志：

```bash
# 首先查看日志大小
docker inspect --format='{{.LogPath}}' versatile-agent
# 输出类似：/var/lib/docker/containers/<id>/<id>-json.log

# 清空日志文件（需要 root 权限，谨慎操作）
sudo truncate -s 0 $(docker inspect --format='{{.LogPath}}' versatile-agent)
```

> 推荐通过 docker-compose 的 logging 配置自动轮转，而不是手动清理。

## 8. 重要提醒：无状态服务特点

adapter-versatile-agent-java 与 EDPAgent 的核心区别：

| 特点 | EDPAgent | adapter-versatile-agent-java |
|------|----------|------------------------------|
| Redis 依赖 | 有，依赖 Redis 存储会话 | **无，不依赖 Redis** |
| 会话状态 | 服务端维护 | 完全通过 A2A 协议传递，无状态 |
| 水平扩展 | 需要 Redis 共享会话 | 可直接水平扩展，加机器即可 |
| 升级/重启 | 需考虑会话丢失 | 随时重启/替换，不影响业务 |

利用无状态特点：
- 可直接重启容器，无需担心会话中断
- 可同时运行多个实例做负载均衡
- 版本升级直接替换容器即可，无需数据迁移
